# KingbaseES V9R2C14 主备集群 / 读写分离集群部署

> 适用版本：KingbaseES V9R2C14
> 参考：https://docs.kingbase.com.cn/cn/KES-V9R2C14/availability/rwc/

---

## 一、集群架构概述

### 1.1 KingbaseES 集群类型

| 架构类型 | 说明 | 适用场景 |
|---------|------|---------|
| **主备架构** | 一主多备，支持自动故障切换 | 高可用场景 |
| **读写分离** | 备节点接受只读查询，分担主节点压力 | 读多写少场景 |
| **同步复制** | 事务提交前确认备机写入 | 数据强一致要求 |
| **异步复制** | 主机写入即返回，性能优先 | 可容忍少量数据丢失 |

### 1.2 集群组件

```
┌─────────────────────────────────────────────────┐
│                   VIP / LVS                       │
│               (读写分离 VIP)                       │
└─────────────────┬───────────────────────────────┘
                  │
     ┌────────────┴────────────┐
     │                         │
┌────▼────┐              ┌────▼────┐
│  Kingbase │              │  Kingbase │
│  Primary  │◄── Streaming ──►│  Standby  │
│  (主库)   │     Replication │  (备库)   │
└─────────┘              └─────────┘
```

## 三、主库配置

### 3.1 初始化主库

```bash
# 在主库节点以 kingbase 用户执行
su - kingbase
cd /opt/Kingbase/ES/V9R2C14/Server/bin

# 初始化主库
./sys_monitor initdb \
  -D /data/kingbase/data \
  -U system \
  -W your_password \
  --encoding=UTF8
```

### 3.2 配置主库 kingbase.conf

```bash
cat > /data/kingbase/data/kingbase.conf << 'EOF'
# === 基础参数 ===
listen_addresses = '*'
port = 54321
max_connections = 1000
superuser_reserved_connections = 10
superuser_reserved_connections = 10

# === 复制参数 ===
wal_level = replica
max_wal_senders = 10
max_replication_slots = 10
wal_keep_size = 1GB
hot_standby = on
max_slot_wal_keep_size = 2GB

# === 归档参数 ===
archive_mode = on
archive_command = 'test ! -f /data/kingbase/wal/%f && cp %p /data/kingbase/wal/%f'
archive_cleanup_command = 'sys_waldump %p > /dev/null 2>&1 || true'

# === 内存参数 ===
shared_buffers = 4GB
effective_cache_size = 12GB
work_mem = 256MB
maintenance_work_mem = 512MB

# === 日志 ===
logging_collector = on
log_directory = 'log'
log_filename = 'kingbase-%Y-%m-%d.log'
log_line_prefix = '%t [%p]: [%l-1] user=%u,db=%d,app=%a,client=%h '
log_statement = 'ddl'
log_min_duration_statement = 1000

# === 性能 ===
wal_buffers = 64MB
checkpoint_completion_target = 0.9
random_page_cost = 1.1
effective_io_concurrency = 200

# === 字符集 ===
client_encoding = UTF8
server_encoding = UTF8

# === 其他 ===
shared_preload_libraries = 'sys_kwr'
track_activity_query_size = 2048
datestyle = 'iso, ymd'
timezone = 'PRC'
EOF
```

### 3.3 配置主库 pg_hba.conf

```bash
cat > /data/kingbase/data/pg_hba.conf << 'EOF'
# TYPE  DATABASE        USER            ADDRESS                 METHOD
# 本地
local   all             system                              trust
local   all             all                                 scram-sha-256
# IPv4 本地复制
host    replication     all             127.0.0.1/32           trust
# 集群内部复制账号（所有备库 IP）
host    replication     all             192.168.1.102/32       trust
host    replication     all             192.168.1.103/32       trust
# 应用连接（所有节点）
host    all             all             192.168.1.0/24          scram-sha-256
EOF
```

### 3.4 创建复制槽（推荐）

```bash
# 在主库创建物理复制槽
./ksql -U system -d test -p 54321 -c "
SELECT * FROM sys_create_physical_replication_slot('standby1_slot', true);
SELECT * FROM sys_create_physical_replication_slot('standby2_slot', true);
"

# 或在 kingbase.conf 中使用 slot_name 参数
# wal_sender_wrapper = 'sys_slot'
```

### 3.5 启动主库

```bash
./sys_monitor start -D /data/kingbase/data
./sys_monitor status -D /data/kingbase/data
```

## 四、备库配置

### 4.1 从主库拉取基础备份

```bash
# 在每个备库节点执行，以 kingbase 用户
su - kingbase
cd /opt/Kingbase/ES/V9R2C14/Server/bin

# 使用 sys_basebackup 拉取备份
./sys_basebackup \
  -h kes-primary \
  -p 54321 \
  -U system \
  -D /data/kingbase/data \
  -R \
  -X stream \
  -P \
  -v

# 参数说明：
# -h : 主库主机名
# -p : 主库端口
# -U : 连接用户
# -D : 备库数据目录
# -R : 自动创建备库标识（创建 standby.signal）
# -X : 复制方式（stream 同步）
# -P : 显示进度
```

### 4.2 备库 kingbase.conf 调整

```bash
# 备库需要修改/确认以下参数
cat >> /data/kingbase/data/kingbase.conf << 'EOF'

# === 备库专用参数 ===
primary_conninfo = 'host=kes-primary port=54321 user=system application_name=standby1'
primary_slot_name = 'standby1_slot'
recovery_target_timeline = 'latest'
hot_standby = on
EOF

# 如果 -R 参数没有生效，手动创建 standby.signal
touch /data/kingbase/data/standby.signal
```

### 4.3 备库 pg_hba.conf

```bash
# 备库只需本地访问
cat > /data/kingbase/data/pg_hba.conf << 'EOF'
local   all             system                              trust
local   all             all                                 scram-sha-256
host    all             all             192.168.1.0/24       scram-sha-256
EOF
```

### 4.4 启动备库

```bash
./sys_monitor start -D /data/kingbase/data
./sys_monitor status -D /data/kingbase/data
```

## 五、验证复制状态

### 5.1 主库端检查

```sql
-- 检查复制连接
SELECT pid, usesysid, usename, application_name, client_addr, client_port, state, sent_lsn, write_lsn, flush_lsn, replay_lsn, sync_state
FROM sys_stat_replication;

-- 检查复制槽
SELECT slot_name, slot_type, active, restart_lsn, confirmed_flush_lsn
FROM sys_replication_slots;

-- 检查 wal 发送进程
SELECT * FROM sys_stat_activity WHERE backend_type = 'walsender';
```

### 5.2 备库端检查

```sql
-- 检查是否为备库
SELECT pg_is_in_recovery();

-- 检查复制状态
SELECT * FROM sys_stat_replication;

-- 检查 wal 接收和应用状态
SELECT pid, application_name, state, sent_lsn, write_lsn, flush_lsn, replay_lsn
FROM sys_stat_wal_receiver;
```

### 5.3 测试数据同步


## 六、读写分离配置

### 6.1 使用 KingbaseES JDBC 读写分离

KingbaseES 的读写分离通过 JDBC 连接串配置实现：

```java
// 读写分离连接串格式
jdbc:kingbase://VIP:54321/database?备库负载均衡参数

// 示例：主库 + 备库的读写分离
jdbc:kingbase://192.168.1.100:54321/testdb?备库权重=1&备库=192.168.1.102:54321
```

### 6.2 常见读写分离参数

| 参数 | 说明 | 示例值 |
|------|------|--------|
| `备库权重` | 备库权重，越高越优先分配 | 1 |
| `备库` | 备库地址列表 | 192.168.1.102:54321 |
| `目标SessionAttrs` | 路由规则 | `targetServerType=primary` |
| `loadBalanceHosts` | 启用负载均衡 | true |

### 6.3 连接串示例

```java
// 配置主库为写，备库为读
String url = "jdbc:kingbase://kes-primary:54321/mydb"
  + "?备库权重=1"
  + "&备库=kes-standby1:54321"
  + "&备库=kes-standby2:54321";

// 强制路由到主库（写操作）
Properties props = new Properties();
props.setProperty("targetServerType", "primary");
Connection conn = DriverManager.getConnection(url, props);

// 自动读写分离（读操作）
props.setProperty("targetServerType", "any");
Connection conn = DriverManager.getConnection(url, props);
```

## 七、自动故障切换（VIP 漂移）

### 7.1 使用 Keepalived 实现 VIP 漂移

```bash
# 所有节点安装 keepalived
yum install -y keepalived

# 主库节点配置
cat > /etc/keepalived/keepalived.conf << 'EOF'
! Configuration File for keepalived
global_defs {
    router_id KES_VIP
}

vrrp_instance VI_KES {
    state BACKUP
    interface eth0
    virtual_router_id 51
    priority 100
    nopreempt
    advert_int 1
    authentication {
        auth_type PASS
        auth_pass 1111
    }
    virtual_ipaddress {
        192.168.1.100/24 dev eth0
    }
    track_script {
        chk_kingbase
    }
}

script_user root
EOF

# 健康检查脚本
cat > /etc/keepalived/chk_kingbase.sh << 'EOF'
#!/bin/bash
result=$(/opt/Kingbase/ES/V9R2C14/Server/bin/sys_monitor status -D /data/kingbase/data 2>&1)
if echo "$result" | grep -q "kingbase (pid"; then
    exit 0
else
    exit 1
fi
EOF
chmod +x /etc/keepalived/chk_kingbase.sh
```

### 7.2 备库降级为主库（手动切换）

```bash
# 在需要提升为主库的节点执行
# 1. 停止当前主库（如果主库已宕机则跳过）
./sys_monitor stop -D /data/kingbase/data -m fast

# 2. 在备库节点执行提升
./sys_monitor promote -D /data/kingbase/data

# 3. 确认已提升为主库
./ksql -U system -d test -p 54321 -c "SELECT pg_is_in_recovery();"

# 4. 更新备库的 primary_conninfo 指向新的主库
```

## 八、常见问题

### 8.1 备库连接不上主库

```bash
# 检查网络
ping kes-primary

# 检查端口
nc -zv kes-primary 54321

# 检查 pg_hba.conf（主库）
# 需要允许备库的 IP 进行 replication 连接
```

### 8.2 复制延迟过大

```sql
-- 主库检查
SELECT pid, usename, application_name, state,
       sent_lsn - replay_lsn AS replication_lag
FROM sys_stat_replication;

-- 可能原因：
-- 1. 网络带宽不足
-- 2. 备库 replay 速度慢（增加备库内存/CPU）
-- 3. wal_keep_size 太小
-- 4. 没有使用复制槽，WAL 被删除
```

### 8.3 备库只读不工作

```bash
# 检查备库配置
grep hot_standby /data/kingbase/data/kingbase.conf
# 应该为 on

# 检查备库当前状态
SELECT pg_is_in_recovery();
SELECT * FROM sys_stat_activity WHERE cardinality(query) > 0;

# 主库检查 max_connections
# 备库 max_connections >= 主库 max_connections
```

```bash
"
```
