# KingbaseES V9R2C14 系统及数据库内存优化

> 本文档介绍 KingbaseES V9R2C14 部署时的系统内核参数和数据库内存参数优化方案。

---

## 一、系统内存优化

### 1.1 查看当前内存使用

```bash
# 查看内存总量
free -h

# 查看详细内存信息
cat /proc/meminfo

# MemTotal:       32766824 kB
# MemFree:        16283424 kB
# MemAvailable:   28000000 kB
# Buffers:          200000 kB
# Cached:          8000000 kB
# SwapCached:            0 kB
# ...

# 查看内存使用分布
ps -eo pid,vsz,rss,pmem,comm --sort=-rss | head -20
```

### 1.2 估算数据库可用内存

```
可用内存 = 物理内存 - 操作系统占用 - 其他应用占用
推荐 shared_buffers = 可用内存 * 25%~40%
推荐 work_mem = (可用内存 - shared_buffers) / (max_connections * 估算并发数)
```

**示例计算（32GB 内存服务器）：**
- 物理内存：32 GB
- 操作系统预留：4 GB
- 其他应用：2 GB
- 可用给 KingbaseES：26 GB
- `shared_buffers`：建议 8~10 GB（约 1/4~1/3 可用内存）
- `work_mem`：剩余 16~18 GB / 预期并发连接数

### 1.3 内核参数配置

```bash
# 编辑 /etc/sysctl.conf
vim /etc/sysctl.conf

# ================== KingbaseES 内核参数 ==================
# 共享内存相关
kernel.shmmax = 17179869184      # 最大共享内存段大小（字节），建议 > shared_buffers
kernel.shmmni = 4096             # 共享内存段最大数量
kernel.shmall = 4194304          # 全部共享内存页数 = shmmax/page_size

# 信号量配置
kernel.sem = 501000 642810000 501000 1280
# 格式：semmsl semmns semopm semmni

# 文件句柄
fs.file-max = 7672460            # 系统级最大文件句柄数

# 异步 I/O
fs.aio-max-nr = 1048576          # 异步 I/O 请求数

# 网络参数
net.ipv4.ip_local_port_range = 9000 65000
net.core.rmem_default = 262144
net.core.rmem_max = 4194304
net.core.wmem_default = 262144
net.core.wmem_max = 4194304

# 内存分配策略（重要！）
vm.overcommit_memory = 0         # 0=启发式, 1=总是, 2=精确
vm.overcommit_ratio = 50         # overcommit_ratio 模式下的最大比例
vm.swappiness = 10               # 交换分区使用倾向（0~100，越低越少使用 swap）

# 使配置生效
sysctl -p
```

### 1.4 vm.overcommit_memory 说明

| 值 | 含义 | 推荐场景 |
|----|------|---------|
| 0 | 启发式 overcommit，操作系统判断 | 默认 |
| 1 | 始终允许 overcommit，不检查 | 不推荐 |
| 2 | 精确检查，总分配 ≤ RAM + swap * overcommit_ratio | **KingbaseES 推荐设为 2** |

**KingbaseES 生产环境建议**：
```bash
echo "vm.overcommit_memory = 2" >> /etc/sysctl.conf
echo "vm.overcommit_ratio = 95" >> /etc/sysctl.conf
sysctl -p
```

### 1.5 禁用大页（Huge Pages）

KingbaseES 不使用大页，建议关闭：

```bash
# 查看当前大页配置
cat /proc/meminfo | grep -i huge

# 临时禁用大页
echo never > /sys/kernel/mm/transparent_hugepage/enabled
echo never > /sys/kernel/mm/transparent_hugepage/defrag

# 永久禁用 - 编辑 /etc/rc.d/rc.local
cat >> /etc/rc.d/rc.local << 'EOF'
if test -f /sys/kernel/mm/transparent_hugepage/enabled; then
    echo never > /sys/kernel/mm/transparent_hugepage/enabled
fi
if test -f /sys/kernel/mm/transparent_hugepage/defrag; then
    echo never > /sys/kernel/mm/transparent_hugepage/defrag
fi
EOF

chmod +x /etc/rc.d/rc.local
```

## 二、数据库内存参数优化

### 2.1 关键内存参数说明

| 参数 | 默认值 | 说明 | 推荐值 |
|------|-------|------|--------|
| `shared_buffers` | 128MB | 共享缓冲区大小 | 物理内存的 25%~40% |
| `work_mem` | 4MB | 单次查询工作内存 | 64MB~256MB |
| `maintenance_work_mem` | 64MB | 维护操作内存 | 512MB~2GB |
| `temp_buffers` | 8MB | 临时缓冲区 | 16MB~64MB |
| `effective_cache_size` | 4MB | 有效缓存大小（查询规划参考） | 物理内存的 75% |
| `bulk_write_ring_factor` | 8 | 批量写入因子 | 64~256 |
| `enable_memoize_hashagg` | on | 哈希聚合缓存 | on |

### 2.2 共享缓冲区配置

```bash
su - kingbase
vim $KINGBASE_DATA/kingbase.conf

# ================== 共享缓冲区配置 ==================
# 32GB 物理内存服务器示例
shared_buffers = 8GB

# 建议值计算：
# 小型服务器（8GB RAM）: shared_buffers = 2GB
# 中型服务器（16GB RAM）: shared_buffers = 4GB
# 大型服务器（32GB RAM）: shared_buffers = 8GB
# 巨型服务器（64GB+ RAM）: shared_buffers = 16GB~32GB
```

### 2.3 工作内存配置

```bash
vim $KINGBASE_DATA/kingbase.conf

# ================== 工作内存配置 ==================
# 单次排序/哈希操作可用内存
work_mem = 128MB

# 维护操作（VACUUM, CREATE INDEX 等）可用内存
maintenance_work_mem = 1GB

# 临时缓冲区
temp_buffers = 64MB

# 有效缓存大小（用于查询规划，不实际分配）
effective_cache_size = 24GB

# 说明：
# work_mem 是每个查询操作（sort/hash/bitmap）可用的内存
# 如果一个查询有多个并发连接，每个连接都可能使用多个 work_mem
# 总 work_mem 消耗 ≈ work_mem * max_connections * 估算并发系数
```

### 2.4 生产环境推荐配置

以下是一个 32GB 内存服务器的完整内存参数配置：

```bash
su - kingbase
vim $KINGBASE_DATA/kingbase.conf

# ================== 内存相关参数 ==================
# 共享缓冲区（总内存的 25%）
shared_buffers = 8GB

# 工作内存（每个操作）
work_mem = 128MB

# 维护内存
maintenance_work_mem = 2GB

# 临时缓冲区
temp_buffers = 64MB

# 有效缓存（用于查询规划）
effective_cache_size = 24GB

# ================== WAL 相关 ==================
wal_buffers = 64MB
wal_writer_delay = 200ms

# ================== 后台写入器 ==================
bgwriter_delay = 200ms
bgwriter_lru_maxpages = 1000
bgwriter_lru_multiplier = 4.0
bgwriter_flush_after = 512kB

# ================== 检查点 ==================
checkpoint_timeout = 15min
checkpoint_completion_target = 0.9
checkpoint_flush_after = 256kB
```

## 三、连接内存配置

### 3.1 连接相关参数

```bash
vim $KINGBASE_DATA/kingbase.conf

# ================== 连接相关 ==================
# 最大连接数（需重启）
max_connections = 500

# 预备语句最大数量
prepared_statements_max = 10000

# 每个连接预留内存估算：
# connection overhead ≈ 6MB~10MB
# 总连接内存 ≈ max_connections * 8MB = 4GB（max_connections=500 时）
```

### 3.2 连接池配置（推荐）

生产环境建议使用 PgBouncer 或 KingbaseES 内置连接池：

```bash
# KingbaseES 内置连接池配置
vim $KINGBASE_DATA/kingbase.conf

# 连接池模式（transaction 或 session）
pool_mode = transaction

# 最大客户端连接
max_client_conn = 1000

# 连接池大小
# 注意：pool_size 必须能被 max_connections 整除
default_pool_size = 100
min_pool_size = 10

# 连接超时
pool_connttl = 3600s
```

## 四、内存优化验证

### 4.1 查看当前内存配置

```bash
# 连接数据库查看所有内存参数
ksql -U system -d test -c "SHOW ALL;" | grep -E "(shared_buffers|work_mem|maint|effective|max_conn)"

# 单独查看
ksql -U system -d test -c "SHOW shared_buffers;"
ksql -U system -d test -c "SHOW work_mem;"
```

### 4.2 监控内存使用

```bash
# 查看 KingbaseES 进程内存使用
ps -eo pid,comm,rss,vsz,pmem --sort=-rss | head -5

# 查看共享内存使用
ipcs -m

# 持续监控（每 5 秒）
watch -n 5 'ps -eo pid,comm,rss,pmem --sort=-rss | head -5'
```

### 4.3 性能诊断查询

```sql
-- 查看当前内存使用情况
SELECT 
    datname,
    numbackends,
    xact_commit,
    xact_rollback,
    blks_hit,
    blks_read,
    stats_reset
FROM sys_stat_database
WHERE datname = 'test';

-- 查看当前活跃连接数和内存使用
SELECT 
    COUNT(*) AS active_connections,
    SUM(usesysid IS NOT NULL)::int AS authenticated
FROM sys_stat_activity;

-- 查看消耗内存最多的查询
SELECT 
    pid,
    now() - query_start AS duration,
    state,
    query
FROM sys_stat_activity
WHERE state != 'idle'
ORDER BY query_start;

-- 查看缓存命中率
SELECT 
    sum(blks_hit) as hits,
    sum(blks_read) as reads,
    round(sum(blks_hit)::numeric / (sum(blks_hit) + sum(blks_read)) * 100, 2) as hit_ratio
FROM sys_stat_database;
```

## 五、Linux 内存诊断

### 5.1 内存问题排查

```bash
# 查看内存使用情况
free -m

# 内存分配详情
cat /proc/meminfo

# 查看哪些进程占用内存最多
ps -eo pid,comm,rss,pmem --sort=-rss | head -20

# 查看 swap 使用
swapon -s
free -h

# 查看 OOM Killer 日志
dmesg | grep -i "out of memory"
journalctl -k | grep -i "out of memory"

# 查看 OOM Killer 配置
cat /proc/`pidof kingbase`/oom_score
cat /proc/`pidof kingbase`/oom_score_adj
```

### 5.2 OOM Killer 配置

如果 KingbaseES 被 OOM Killer 杀死，可以调整优先级：

```bash
# 降低 OOM Killer 优先级的可能性
echo -1000 > /proc/`pidof kingbase`/oom_score_adj

# 或者在启动脚本中设置
vim /opt/KingbaseES/V9R2C14/bin/kingbase.service
# 在 [Service] 段添加：
# OOMScoreAdjust=-1000
```

### 5.3 透明大页对性能影响

```bash
# 检查透明大页状态
cat /sys/kernel/mm/transparent_hugepage/enabled
# [always] madvise never  <- always 表示启用

# 透明大页可能导致 KingbaseES 内存分配变慢
# 建议始终关闭
echo never > /sys/kernel/mm/transparent_hugepage/enabled
echo never > /sys/kernel/mm/transparent_hugepage/defrag
```

## 六、优化脚本

### 6.1 一键优化脚本

```bash
#!/bin/bash
# kingbase_memory_tune.sh - KingbaseES 内存优化脚本

cat > /etc/sysctl.d/99-kingbase.conf << 'EOF'
# KingbaseES 内核参数优化
kernel.shmmax = 17179869184
kernel.shmmni = 4096
kernel.shmall = 4194304
kernel.sem = 501000 642810000 501000 1280
fs.file-max = 7672460
fs.aio-max-nr = 1048576
net.ipv4.ip_local_port_range = 9000 65000
net.core.rmem_default = 262144
net.core.rmem_max = 4194304
net.core.wmem_default = 262144
net.core.wmem_max = 4194304
vm.overcommit_memory = 2
vm.overcommit_ratio = 95
vm.swappiness = 10
EOF

# 使配置生效
sysctl -p /etc/sysctl.d/99-kingbase.conf

# 禁用透明大页
cat > /etc/rc.d/rc.local << 'EOF'
if test -f /sys/kernel/mm/transparent_hugepage/enabled; then
    echo never > /sys/kernel/mm/transparent_hugepage/enabled
fi
if test -f /sys/kernel/mm/transparent_hugepage/defrag; then
    echo never > /sys/kernel/mm/transparent_hugepage/defrag
fi
EOF

chmod +x /etc/rc.d/rc.local
/etc/rc.d/rc.local

echo "KingbaseES 内存优化完成"
```

## 七、常见问题

### 7.1 内存使用过高

**症状**：`KingbaseES` 进程内存持续增长

**排查**：
```sql
-- 查看长时间运行的查询
SELECT pid, now() - query_start AS running_time, state, query
FROM sys_stat_activity
WHERE state != 'idle'
ORDER BY query_start;

-- 查看是否有大量临时文件生成
SELECT pid, query, temp_files, temp_bytes
FROM sys_stat_activity
JOIN sys_database ON datid = oid
WHERE temp_files > 0
ORDER BY temp_bytes DESC;
```

### 7.2 Swap 被大量使用

**解决**：
1. 减小 `shared_buffers`（但可能降低性能）
2. 减小 `max_connections`
3. 增加物理内存
4. 调整 `vm.swappiness = 5` 或更低

### 7.3 缓存命中率低

**解决**：
1. 增加 `shared_buffers`
2. 优化 SQL 查询
3. 调整 `effective_cache_size` 使查询规划更准确
4. 检查是否有大量全表扫描
