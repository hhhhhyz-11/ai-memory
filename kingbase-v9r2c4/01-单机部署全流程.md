# KingbaseES V9R2C14 单机部署全流程

> 本文档详细介绍 KingbaseES V9R2C14 单机环境的完整部署流程，包括系统准备、安装配置、初始化和验证。

---

## 一、系统环境准备

### 1.1 系统要求

| 项目 | 最低要求 | 推荐配置 |
|------|---------|---------|
| CPU | 4 核 | 8 核及以上 |
| 内存 | 8 GB | 16 GB 及以上 |
| 磁盘 | 100 GB | 200 GB SSD |
| 操作系统 | CentOS 7.6+ / RHEL 7+ / 麒麟 V10 / 统信UOS | 同左 |

### 1.2 操作系统检查

```bash
# 检查操作系统版本
cat /etc/os-release
# CentOS Linux release 7.9.2009 (Core)

# 检查 CPU 核心数
grep "processor" /proc/cpuinfo | wc -l

# 检查内存
free -h
#               total        used        free      shared  buff/cache   available
# Mem:           31Gi       2.1Gi        26Gi       300Mi       2.5Gi        28Gi

# 检查磁盘空间
df -h /
# Filesystem      Size  Used Avail Use% Mounted on
# /dev/sda1       100G   50G   50G  50% /

# 检查 hostname
hostname
```

### 1.3 关闭防火墙

```bash
# CentOS 7 / RHEL 7
systemctl stop firewalld
systemctl disable firewalld

# 验证防火墙状态
systemctl status firewalld
# 或
firewall-cmd --state
```

### 1.4 关闭 SELinux

```bash
# 查看 SELinux 状态
getenforce
# Disabled

# 临时关闭
setenforce 0

# 永久关闭 - 编辑 /etc/selinux/config
sed -i 's/SELINUX=enforcing/SELINUX=disabled/g' /etc/selinux/config

# 重启后生效
reboot
```

### 1.5 设置系统参数

```bash
# 编辑 /etc/sysctl.conf，添加以下内容
cat >> /etc/sysctl.conf << 'EOF'
# KingbaseES 所需内核参数
kernel.shmmax = 68719476736
kernel.shmmni = 4096
kernel.shmall = 4294967296
kernel.sem = 501000 642810000 501000 1280
fs.file-max = 7672460
fs.aio-max-nr = 1048576
net.ipv4.ip_local_port_range = 9000 65000
net.core.rmem_default = 262144
net.core.rmem_max = 4194304
net.core.wmem_default = 262144
net.core.wmem_max = 4194304
vm.overcommit_memory = 0
vm.swappiness = 10
EOF

# 使配置生效
sysctl -p

# 编辑 /etc/security/limits.conf
cat >> /etc/security/limits.conf << 'EOF'
# KingbaseES 用户资源限制
kingbase soft nofile 65536
kingbase hard nofile 65536
kingbase soft nproc 65536
kingbase hard nproc 65536
kingbase soft core unlimited
kingbase hard core unlimited
kingbase soft memlock unlimited
kingbase hard memlock unlimited
EOF
```

### 1.6 创建数据库用户

```bash
# 创建 kingbase 用户组
groupadd kingbase

# 创建 kingbase 用户
useradd -g kingbase -m -d /home/kingbase -s /bin/bash kingbase

# 设置密码
echo "kingbase:Kingbase123" | chpasswd

# 验证用户
id kingbase
# uid=1001(kingbase) gid=1001(kingbase) groups=1001(kingbase)
```

### 1.7 创建安装目录

```bash
# 创建 KingbaseES 安装目录
mkdir -p /opt/KingbaseES/V9R2C14
mkdir -p /data/kingbase/data
mkdir -p /data/kingbase/wal

# 授权
chown -R kingbase:kingbase /opt/KingbaseES
chown -R kingbase:kingbase /data/kingbase

# 验证目录权限
ls -ld /opt/KingbaseES /data/kingbase
```

## 二、KingbaseES 安装

### 2.1 上传安装包

```bash
# 使用 scp 或其他方式上传安装包到服务器
# 安装包名称示例：KingbaseES_V9R2C14_centos7_x86_64.tar

# 解压安装包
cd /opt/KingbaseES
tar -xf KingbaseES_V9R2C14_centos7_x86_64.tar

# 查看解压内容
ls -la
```

### 2.2 图形化安装（VNC 桌面环境）

如果使用图形化界面安装：

```bash
# 切换到 kingbase 用户（图形界面需以 kingbase 用户运行）
su - kingbase

# 进入安装目录
cd /opt/KingbaseES/KingbaseES

# 启动安装向导
./setup.sh
```

安装向导步骤：
1. 选择安装语言（中文）
2. 阅读并接受许可协议
3. 选择安装组件（完整安装）
4. 选择安装路径：`/opt/KingbaseES/V9R2C14`
5. 配置数据库端口（默认 54321）
6. 设置数据库超级用户密码
7. 选择数据库初始化选项

### 2.3 命令行静默安装

```bash
# 切换到 root，准备配置文件
su - root

# 创建安装配置文件
cat > /tmp/kingbase_install.xml << 'EOF'
<?xml version="1.0"?>
<Install>
    <Language>zh_CN</Language>
    <InstallMode>typical</InstallMode>
    <InstallPath>/opt/KingbaseES/V9R2C14</InstallPath>
    <DataPath>/data/kingbase/data</DataPath>
    <Port>54321</Port>
    <SuperUser>system</SuperUser>
    <Password>YourStrongPassword123</Password>
    <Locale>zh_CN.UTF-8</Locale>
</Install>
EOF

# 以 kingbase 用户执行安装
su - kingbase -c "cd /opt/KingbaseES/KingbaseES && ./setup.sh -f /tmp/kingbase_install.xml"
```

### 2.4 安装目录结构

```bash
ls -la /opt/KingbaseES/V9R2C14/
# bin/          - 可执行文件
# include/      - 头文件
# lib/          - 库文件
# share/        - 共享文件
# docs/         - 文档
# JRE/          - Java 运行环境
# Uninstall/    - 卸载程序
```

## 三、数据库初始化

### 3.1 配置环境变量

```bash
# 以 kingbase 用户编辑 .bash_profile
su - kingbase
cat >> ~/.bash_profile << 'EOF'
# KingbaseES Environment
export KINGBASE_HOME=/opt/KingbaseES/V9R2C14
export PATH=$KINGBASE_HOME/bin:$PATH
export LD_LIBRARY_PATH=$KINGBASE_HOME/lib:$LD_LIBRARY_PATH
export KINGBASE_DATA=/data/kingbase/data
export KINGBASE_PORT=54321
EOF

# 使环境变量生效
source ~/.bash_profile

# 验证环境变量
echo $KINGBASE_HOME
echo $PATH | grep kingbase
```

### 3.2 初始化数据库

```bash
# 切换到 kingbase 用户
su - kingbase

# 初始化数据库（如果 data 目录为空）
initdb -D $KINGBASE_DATA -U system -W --encoding=UTF8 --locale=zh_CN.UTF-8

# 或者使用 kingbase init 命令
sys_ctl initdb -D $KINGBASE_DATA -U system -W
```

### 3.3 配置数据库参数

编辑 `$KINGBASE_DATA/kingbase.conf`：

```bash
# 编辑配置文件
vim $KINGBASE_DATA/kingbase.conf

# ================== 关键配置项 ==================

# 监听地址（生产环境建议绑定内网IP）
listen_addresses = '*'
# 或
listen_addresses = '192.168.1.100'

# 端口
port = 54321

# 最大连接数
max_connections = 500

# 共享缓冲区（建议为系统内存的 1/4）
shared_buffers = 4GB

# 工作内存
work_mem = 64MB

# 维护工作内存
maintenance_work_mem = 512MB

#  WAL 日志大小
wal_level = replica
max_wal_size = 2GB
min_wal_size = 80MB

# 归档配置（可选）
archive_mode = on
archive_command = 'test ! -f /data/kingbase/wal/%f && cp %p /data/kingbase/wal/%f'

# 日志配置
logging_collector = on
log_directory = 'log'
log_filename = 'kingbase-%Y-%m-%d_%H%M%S.log'
log_rotation_age = 1d
log_rotation_size = 100MB
log_line_prefix = '%t [%p]: [%l-1] user=%u,db=%d,app=%a,client=%h '

# 时区
timezone = 'Asia/Shanghai'

# 字符集
client_encoding = UTF8
server_encoding = UTF8

# 允许远程连接 - 编辑 sys_hba.conf
# host all all 0.0.0.0/0 md5
```

### 3.4 配置远程访问（pg_hba.conf）

编辑 `$KINGBASE_DATA/sys_hba.conf`：

```bash
vim $KINGBASE_DATA/sys_hba.conf

# 添加以下内容（根据实际网段调整）
# TYPE  DATABASE        USER            ADDRESS                 METHOD

# 本地连接
local   all             system                                 trust
local   all             kingbase                               trust

# IPv4 本地回环
host    all             all             127.0.0.1/32            md5

# 允许内网段访问
host    all             all             192.168.1.0/24          md5

# 允许所有 IP 访问（测试环境）
host    all             all             0.0.0.0/0               md5

# 允许 IPv6
host    all             all             ::/0                    md5
```

### 3.5 启动数据库

```bash
# 切换到 kingbase 用户
su - kingbase

# 启动数据库
sys_ctl -D $KINGBASE_DATA start

# 或者使用 kingbase 命令
kingbase start -D $KINGBASE_DATA

# 查看启动日志
tail -f $KINGBASE_DATA/log/*.log
```

### 3.6 验证数据库运行状态

```bash
# 检查进程
ps -ef | grep kingbase | grep -v grep
# kingbase  12345     1  ... /opt/KingbaseES/V9R2C14/bin/kingbase -D /data/kingbase/data

# 检查端口
netstat -tlnp | grep 54321
# tcp        0      0 0.0.0.0:54321           0.0.0.0:*               LISTEN      12345/kingbase

# 测试本地连接
ksql -U system -d test
# 或者指定端口
ksql -U system -d test -p 54321

# 查看数据库版本
ksql -U system -d test -c "SELECT version();"
```

## 四、数据库基本操作

### 4.1 连接数据库

```bash
# 本地连接
ksql -U system -d test

# 远程连接
ksql -U system -d test -h 192.168.1.100 -p 54321

# 连接时指定编码
ksql -U system -d test -E UTF8
```

### 4.2 常用 SQL 操作

```sql
-- 查看所有数据库
SELECT datname FROM sys_database;

-- 查看当前数据库
SELECT current_database();

-- 查看所有用户/角色
SELECT rolname FROM sys_roles;

-- 创建数据库
CREATE DATABASE mydb OWNER system ENCODING 'UTF8';

-- 创建用户
CREATE USER myuser WITH PASSWORD 'MyPassword123';
GRANT ALL PRIVILEGES ON DATABASE mydb TO myuser;

-- 授权
GRANT ALL PRIVILEGES ON ALL TABLES IN SCHEMA public TO myuser;
GRANT ALL PRIVILEGES ON ALL SEQUENCES IN SCHEMA public TO myuser;
```

### 4.3 启停服务命令

```bash
# 启动
sys_ctl -D $KINGBASE_DATA start

# 停止（快速停止）
sys_ctl -D $KINGBASE_DATA stop -m fast

# 停止（优雅等待连接断开）
sys_ctl -D $KINGBASE_DATA stop -m smart

# 立即停止（不推荐）
sys_ctl -D $KINGBASE_DATA stop -m immediate

# 重启
sys_ctl -D $KINGBASE_DATA restart

# 重新加载配置
sys_ctl -D $KINGBASE_DATA reload

# 检查配置
sys_ctl -D $KINGBASE_DATA show all
```

## 五、常见问题处理

### 5.1 安装报错

**问题**：安装时报错 `bash: ./setup.sh: /lib/ld-linux.so.2: bad ELF interpreter`

**解决**：安装 32 位库
```bash
yum install glibc.i686 libstdc++.i686 -y
```

### 5.2 数据库无法启动

**问题**：`could not create shared memory segment`

**解决**：
```bash
# 增加共享内存
vim /etc/sysctl.conf
# kernel.shmmax = 68719476736
# kernel.shmall = 4294967296
sysctl -p
```

### 5.3 连接被拒绝

**问题**：`could not connect to server: Connection refused`

**解决**：
1. 检查端口是否正确：`netstat -tlnp | grep 54321`
2. 检查 `kingbase.conf` 中 `listen_addresses` 配置
3. 检查 `sys_hba.conf` 是否允许该 IP

### 5.4 忘记 system 用户密码

**解决**（单用户模式重置）：
```bash
# 停止数据库
sys_ctl -D $KINGBASE_DATA stop

# 单用户模式启动
kingbase -D $KINGBASE_DATA -p 54321

# 在单用户模式下修改密码
# ALTER USER system WITH PASSWORD 'NewPassword123';
# \q 退出
```
