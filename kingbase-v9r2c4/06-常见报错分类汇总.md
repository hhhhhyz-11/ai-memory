# KingbaseES V9R2C14 常见报错分类汇总

> 本文档汇总 KingbaseES V9R2C14 使用过程中常见的错误，按类别分类并提供解决方案。

---

## 一、连接类错误

### 1.1 连接被拒绝

**错误信息**：
```
could not connect to server: Connection refused
Is the server running on host "192.168.1.100" and accepting
TCP/IP connections on port 54321?
```

**可能原因**：
1. 数据库服务未启动
2. 端口配置不正确
3. 防火墙阻止
4. `listen_addresses` 未正确配置

**排查步骤**：

```bash
# 1. 检查数据库进程是否运行
ps -ef | grep kingbase | grep -v grep

# 2. 检查端口监听
netstat -tlnp | grep 54321
# 或
ss -tlnp | grep 54321

# 3. 检查防火墙
systemctl status firewalld
firewall-cmd --list-all

# 4. 检查 kingbase.conf 中的监听配置
grep "listen_addresses" $KINGBASE_DATA/kingbase.conf
grep "^port" $KINGBASE_DATA/kingbase.conf
```

**解决方案**：

```bash
# 启动数据库
sys_ctl -D $KINGBASE_DATA start

# 修改 kingbase.conf
vim $KINGBASE_DATA/kingbase.conf
# listen_addresses = '*'  或  listen_addresses = '0.0.0.0'
# port = 54321

# 重新加载配置
sys_ctl -D $KINGBASE_DATA reload

# 开放防火墙端口（如果需要）
firewall-cmd --permanent --add-port=54321/tcp
firewall-cmd --reload
```

### 1.2 认证失败

**错误信息**：
```
FATAL: password authentication failed for user "system"
FATAL: no pg_hba.conf entry for the connection
```

**可能原因**：
1. 用户名或密码错误
2. `sys_hba.conf` 中没有对应条目

**排查步骤**：

```bash
# 检查 sys_hba.conf 配置
grep -v "^#" $KINGBASE_DATA/sys_hba.conf | grep -v "^$"

# 确认连接方式和认证方法
# TYPE  DATABASE        USER            ADDRESS                 METHOD
# host    all             all             0.0.0.0/0               md5
```

**解决方案**：

```bash
# 修改 sys_hba.conf 添加允许条目
vim $KINGBASE_DATA/sys_hba.conf
# host    all             all             192.168.1.0/24          md5

# 重新加载配置
sys_ctl -D $KINGBASE_DATA reload

# 或修改用户密码
ksql -U system -d test -h localhost -c "ALTER USER system WITH PASSWORD 'NewPassword123';"
```

---

### 1.3 连接数过多

**错误信息**：
```
FATAL: remaining connection slots reserved for non-replication superuser connections
FATAL: too many clients already
```

**可能原因**：
1. `max_connections` 设置过小
2. 连接泄漏（应用未关闭连接）
3. 连接池配置不当

**排查步骤**：

```sql
-- 查看当前连接数
SELECT 
    COUNT(*) AS current_connections,
    MAX(max_connections) AS max_connections
FROM sys_stat_database;

-- 查看各用户连接数
SELECT 
    usename,
    COUNT(*) AS connections
FROM sys_stat_activity
GROUP BY usename
ORDER BY COUNT(*) DESC;

-- 查看最长运行的查询
SELECT 
    pid,
    usename,
    client_addr,
    now() - query_start AS duration,
    state,
    query
FROM sys_stat_activity
WHERE state != 'idle'
ORDER BY query_start;
```

**解决方案**：

```bash
# 方法一：杀掉空闲连接
# 在 SQL 中执行
SELECT sys_terminate_backend(pid)
FROM sys_stat_activity
WHERE state = 'idle'
AND query_start < now() - interval '30 minutes';

# 方法二：增加 max_connections（需重启）
vim $KINGBASE_DATA/kingbase.conf
# max_connections = 1000

sys_ctl -D $KINGBASE_DATA restart

# 方法三：使用连接池
# 配置 PgBouncer 或 KingbaseES 内置连接池
```

## 二、启动类错误

### 2.1 共享内存不足

**错误信息**：
```
FATAL: could not create shared memory segment: Cannot allocate memory
DETAIL: Failed system call was shmget(key=5432001, size=..., ...
```

**可能原因**：
1. 内核 `shmmax` 或 `shmall` 参数设置过小
2. 物理内存不足

**解决方案**：

```bash
# 检查当前共享内存配置
ipcs -m

# 查看内核参数
cat /proc/sys/kernel/shmmax
cat /proc/sys/kernel/shmall

# 修改内核参数
vim /etc/sysctl.conf
# kernel.shmmax = 17179869184   (16GB)
# kernel.shmall = 4194304

sysctl -p

# 或临时修改
sysctl -w kernel.shmmax=17179869184
sysctl -w kernel.shmall=4194304

# 检查物理内存
free -h
```

---

### 2.2 数据目录不存在或无权限

**错误信息**：
```
FATAL: data directory "/data/kingbase/data" does not exist
FATAL: could not open permissions lock file
```

**解决方案**：

```bash
# 创建数据目录
mkdir -p /data/kingbase/data
mkdir -p /data/kingbase/wal

# 设置权限
chown -R kingbase:kingbase /data/kingbase

# 检查权限
ls -ld /data/kingbase/data
```

---

### 2.3 WAL 日志损坏

**错误信息**：
```
FATAL: WAL could not be initialized
FATAL: could not open file "...": No such file or directory
```

**解决方案**：

```bash
# 如果有备份，从备份恢复
# 或重新初始化

# 备份损坏的 WAL
mv /data/kingbase/data/pg_wal /data/kingbase/data/pg_wal.bak.$(date +%Y%m%d)

# 创建新的 WAL 目录
mkdir /data/kingbase/data/pg_wal
chown kingbase:kingbase /data/kingbase/data/pg_wal

# 启动数据库
sys_ctl -D $KINGBASE_DATA start
```

## 三、SQL 执行类错误

### 3.1 语法错误

**错误信息**：
```
ERROR: syntax error at or near "..."
```

**解决方案**：
1. 检查 SQL 语句语法
2. 确认关键字大小写（KingbaseES 关键字通常不区分大小写）
3. 检查引号使用（双引号用于标识符，单引号用于字符串）

```sql
-- 正确示例
SELECT * FROM "User" WHERE name = 'test';

-- 错误示例
SELEC * FORM users;  -- 关键字拼写错误
```

---

### 3.2 数据类型不匹配

**错误信息**：
```
ERROR: operator does not exist: integer = text
LINE 1: SELECT * FROM table WHERE id = '1';
```

**解决方案**：
```sql
-- 使用正确的类型转换
SELECT * FROM table WHERE id = 1;

-- 或显式转换
SELECT * FROM table WHERE id = CAST('1' AS INTEGER);
SELECT * FROM table WHERE id = '1'::INTEGER;
```

---

### 3.3 主键/唯一键冲突

**错误信息**：
```
ERROR: duplicate key value violates unique constraint "xxx_pkey"
DETAIL: Key (id)=(1) already exists.
```

**排查与解决**：

```sql
-- 查看冲突数据
SELECT * FROM table_name WHERE id = 1;

-- 查看当前最大 ID
SELECT MAX(id) FROM table_name;

-- 使用 ON CONFLICT 处理
INSERT INTO table_name (id, name) VALUES (1, 'test')
ON CONFLICT (id) DO UPDATE SET name = EXCLUDED.name;

-- 或使用序列重置
SELECT setval('table_name_id_seq', (SELECT MAX(id) FROM table_name));
```

---

### 3.4 外键约束失败

**错误信息**：
```
ERROR: insert or update on table "child" violates foreign key constraint "fk_parent"
DETAIL: Key (parent_id)=(999) is not present in the parent table "parent".
```

**解决方案**：
1. 先插入父表记录
2. 或删除冲突的外键约束
3. 或插入正确的外键值

## 四、权限类错误

### 4.1 权限不足

**错误信息**：
```
ERROR: permission denied for relation "xxx"
ERROR: must be owner of table "xxx"
```

**解决方案**：

```sql
-- 授予表权限
GRANT SELECT, INSERT, UPDATE, DELETE ON TABLE xxx TO username;

-- 授予所有表权限
GRANT ALL PRIVILEGES ON ALL TABLES IN SCHEMA public TO username;

-- 授予序列权限（自增主键需要）
GRANT ALL PRIVILEGES ON ALL SEQUENCES IN SCHEMA public TO username;

-- 授予架构权限
GRANT USAGE ON SCHEMA public TO username;

-- 设置默认权限
ALTER DEFAULT PRIVILEGES IN SCHEMA public GRANT ALL ON TABLES TO username;
ALTER DEFAULT PRIVILEGES IN SCHEMA public GRANT ALL ON SEQUENCES TO username;
```

---

### 4.2 无访问权限

**错误信息**：
```
ERROR: permission denied for database "xxx"
```

**解决方案**：

```sql
-- 授予数据库权限
GRANT CONNECT ON DATABASE xxx TO username;
GRANT TEMPORARY ON DATABASE xxx TO username;
```

## 五、备份恢复类错误

### 5.1 备份文件损坏

**错误信息**：
```
pg_restore: [archiver] unsupported version (1.14) in file header
```

**解决方案**：
1. 使用与目标库版本匹配的 pg_dump 版本
2. 重新执行备份

---

### 5.2 恢复时表空间不存在

**错误信息**：
```
ERROR: tablespace "xxx" does not exist
```

**解决方案**：

```sql
-- 创建表空间
CREATE TABLESPACE xxx LOCATION '/path/to/tablespace';

-- 或在恢复时跳过表空间
pg_restore -d dbname --no-tablespaces dumpfile.tar
```

## 六、复制类错误

### 6.1 复制槽不存在

**错误信息**：
```
FATAL: replication slot "xxx" does not exist
```

**解决方案**：

```sql
-- 在主库创建复制槽
SELECT * FROM sys_create_physical_replication_slot('slot_name');

-- 查看现有复制槽
SELECT * FROM sys_replication_slots;

-- 查看备库使用的槽名
-- primary_slot_name = 'slot_name'
```

---

### 6.2 复制延迟过大

**错误信息**：
（无错误，但备库数据滞后）

**排查与解决**：

```sql
-- 主库查看复制状态
SELECT 
    pid,
    usename,
    application_name,
    client_addr,
    state,
    sent_lsn,
    write_lsn,
    flush_lsn,
    replay_lsn,
    sent_lsn - replay_lsn AS delay_bytes
FROM sys_stat_replication;

-- 查看备库延迟
SELECT 
    now() - pg_last_xact_replay_timestamp() AS replay_delay;
```

**优化方案**：
```bash
# 主库优化
vim $KINGBASE_DATA/kingbase.conf
# wal_level = replica
# wal_keep_size = 128
# max_wal_senders = 5

# 增加 wal_writer_delay
# 或使用复制槽替代保留 WAL 文件
```

## 七、日志查看与分析

### 7.1 日志位置

```bash
# 默认日志位置
ls -la $KINGBASE_DATA/log/
ls -la $KINGBASE_HOME/log/

# 查看最近错误
grep -i error $KINGBASE_DATA/log/*.log | tail -50
grep -i fatal $KINGBASE_DATA/log/*.log | tail -50
```

### 7.2 常用日志分析

```bash
# 查看启动日志
cat $KINGBASE_DATA/log/*.log | grep -i "started\|listen\|ready"

# 查看连接日志
grep "connection received" $KINGBASE_DATA/log/*.log

# 查看错误日志
grep -i "error\|fatal\|panic" $KINGBASE_DATA/log/*.log | tail -100

# 查看检查点
grep "checkpoint" $KINGBASE_DATA/log/*.log | tail -20
```

### 7.3 开启详细日志

```bash
vim $KINGBASE_DATA/kingbase.conf

# 开启查询日志
logging_collector = on
log_directory = 'log'
log_filename = 'kingbase-%Y-%m-%d.log'

# 记录所有 SQL
log_statement = 'all'

# 记录慢查询
log_min_duration_statement = 1000  # 记录超过 1 秒的查询

# 记录连接断开
log_disconnections = on
log_connections = on

# 重载配置
sys_ctl -D $KINGBASE_DATA reload
```

## 八、错误代码速查

| 错误代码 | 说明 | 解决方向 |
|---------|------|---------|
| 28P01 | 认证失败 | 检查密码 |
| 3D000 | 数据库不存在 | 检查数据库名 |
| 42P01 | 表不存在 | 检查表名 |
| 23505 | 主键冲突 | 检查唯一约束 |
| 23503 | 外键约束失败 | 检查外键关系 |
| 42501 | 权限不足 | 授权 |
| 53000 | 资源不足 | 增加资源或优化 |
| 08006 | 连接失败 | 检查网络和服务 |
| 57P01 | Admin shutdown | 数据库被管理性关闭 |
| 08003 | 连接不存在 | 应用连接问题 |
