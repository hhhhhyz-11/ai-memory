# KingbaseES V9R2C14 断点续传配置

> 本文档介绍在数据迁移和 KFS 同步场景下的断点续传配置与使用方法。

---

## 一、断点续传概述

### 1.1 什么是断点续传

断点续传（Checkpoint Resume）是指在数据传输过程中，当发生网络中断、服务重启等异常情况时，能够从上次中断的位置继续传输，而不需要重新开始。

### 1.2 应用场景

| 场景 | 说明 |
|------|------|
| KDTS 迁移 | 大数据量迁移时避免重复传输 |
| KFS 同步 | 网络不稳定环境下的文件同步 |
| 备份恢复 | 大型数据库备份中断后继续 |
| 数据抽取 | ETL 过程中断后继续 |

## 二、KDTS 断点续传配置

### 2.1 KDTS 迁移断点续传原理

KDTS 通过记录已迁移的数据位置，在中断后能够从断点处继续：

```
迁移进度: 0% → 50% → [中断] → 50% → 100%
                    ↑
                 从此位置继续
```

### 2.2 配置断点续传

```bash
# 创建带断点续传配置的迁移任务
cat > /opt/kdts_jobs/migration_with_checkpoint.json << 'EOF'
{
    "job": {
        "name": "full_migration_with_checkpoint",
        "type": "FULL",
        "source": {
            "type": "ORACLE",
            "host": "192.168.1.10",
            "port": 1521,
            "serviceName": "ORCL",
            "username": "SRC_USER",
            "password": "SrcPassword123"
        },
        "target": {
            "type": "KINGBASE",
            "host": "192.168.1.100",
            "port": 54321,
            "database": "test",
            "username": "kdts_user",
            "password": "KdtsPassword123"
        },
        "checkpoint": {
            "enabled": true,
            "interval": 1000,
            "storage": "table",
            "table_name": "kdts_checkpoint",
            "auto_resume": true
        },
        "options": {
            "parallel": 4,
            "batchSize": 10000,
            "commitSize": 50000
        }
    }
}
EOF
```

### 2.3 手动管理断点

```bash
# 查看当前断点状态
$KINGBASE_HOME/../kdts/bin/kdts-cli.sh -checkpoint-status -job full_migration_with_checkpoint

# 手动设置断点（从头开始）
$KINGBASE_HOME/../kdts/bin/kdts-cli.sh -checkpoint-reset -job full_migration_with_checkpoint

# 从指定位置恢复
$KINGBASE_HOME/../kdts/bin/kdts-cli.sh -checkpoint-resume -job full_migration_with_checkpoint -from 50000
```

### 2.4 断点表结构

KDTS 在目标库创建断点记录表：

```sql
-- 断点记录表
CREATE TABLE kdts_checkpoint (
    job_id VARCHAR2(100),
    table_name VARCHAR2(200),
    last_key VARCHAR2(500),
    last_offset BIGINT,
    status VARCHAR2(20),
    update_time TIMESTAMP DEFAULT NOW(),
    PRIMARY KEY (job_id, table_name)
);

-- 查看断点记录
SELECT * FROM kdts_checkpoint WHERE job_id = 'full_migration_with_checkpoint';
```

### 2.5 异常中断后的恢复

```bash
# 1. 检查上次任务状态
$KINGBASE_HOME/../kdts/bin/kdts-cli.sh -status -job full_migration_with_checkpoint

# 2. 如果状态为 FAILED/INTERRUPTED，从断点恢复
$KINGBASE_HOME/../kdts/bin/kdts-cli.sh -resume -job full_migration_with_checkpoint

# 3. 或使用自动恢复
$KINGBASE_HOME/../kdts/bin/kdts-cli.sh -config /opt/kdts_jobs/migration_with_checkpoint.json -auto_resume

# 4. 确认恢复进度
$KINGBASE_HOME/../kdts/bin/kdts-cli.sh -progress -job full_migration_with_checkpoint
```

## 三、KFS 断点续传配置

### 3.1 KFS 断点续传原理

KFS 通过记录已同步的文件列表和校验信息，在中断后能够跳过已同步的文件：

```
已同步: file_001.txt, file_002.txt, [中断] file_003.txt, file_004.txt
                          ↑
                      从此文件继续
```

### 3.2 配置 KFS 断点续传

```bash
vim $KINGBASE_HOME/kfs/config/kfs_checkpoint.conf

# ================== KFS 断点续传配置 ==================

# 集群标识
cluster.name = kfs_checkpoint_cluster
node.id = kfs_node_01
node.role = sender

# 网络配置
node.host = 192.168.1.101
node.port = 8891

# 目标端配置
target.1.host = 192.168.1.102
target.1.port = 8891

# 断点续传配置
checkpoint.enabled = true
checkpoint.interval = 1000
checkpoint.storage = file
checkpoint.file = /data/kingbase/kfs/checkpoint/kfs_checkpoint.dat

# 断点信息保留策略
checkpoint.retention = 7d
checkpoint.compression = on

# 同步模式
sync.mode = file
sync.file.checkpoint = true
sync.checksum = md5

# KingbaseES 连接
kingbase.host = localhost
kingbase.port = 54321
kingbase.database = test
kingbase.user = system
kingbase.password = YourPassword123
```

### 3.3 断点文件目录

```bash
# 创建断点文件存储目录
mkdir -p /data/kingbase/kfs/checkpoint
chown -R kingbase:kingbase /data/kingbase/kfs

# 查看断点文件
ls -la /data/kingbase/kfs/checkpoint/
```

### 3.4 断点续传命令

```bash
# 查看当前断点状态
$KINGBASE_HOME/../kfs/bin/kfs_admin.sh checkpoint-status

# 列出已同步的文件
$KINGBASE_HOME/../kfs/bin/kfs_admin.sh list-checkpoint-files

# 清除断点（从头开始）
$KINGBASE_HOME/../kfs/bin/kfs_admin.sh checkpoint-clear

# 手动触发断点保存
$KINGBASE_HOME/../kfs/bin/kfs_admin.sh checkpoint-save

# 从断点恢复同步
$KINGBASE_HOME/../kfs/bin/kfs_admin.sh resume
```

### 3.5 验证断点续传

```bash
# 1. 模拟同步中断
# （停止 KFS 服务）
$KINGBASE_HOME/../kfs/bin/kfs_admin.sh stop

# 2. 查看断点状态
$KINGBASE_HOME/../kfs/bin/kfs_admin.sh checkpoint-status
# 应显示：Last synced file: file_050.txt, offset: 1024000

# 3. 重新启动 KFS
$KINGBASE_HOME/../kfs/bin/kfs_server --config $KINGBASE_HOME/kfs/config/kfs_checkpoint.conf --role sender -d

# 4. 确认从断点恢复
tail -f /data/kingbase/kfs/log/kfs_sender.log
# 应显示：Resuming from checkpoint: file_050.txt
```

## 四、备份恢复断点续传

### 4.1 Base Backup 断点续传

```bash
# KingbaseES 不支持直接断点续传 base backup
# 但可以通过以下方式实现：

# 方案一：使用 rsync 增量备份（支持断点续传）
rsync -avz --partial --progress \
    kingbase@source:/data/kingbase/data/ \
    /data/kingbase/data/

# --partial: 保留部分传输的文件
# --progress: 显示传输进度

# 方案二：使用 tar + nc 网络传输（不支持断点续传）
# 需要使用其他工具实现断点

# 方案三：使用专业备份工具（如 Barman）
```

### 4.2 使用 rsync 实现数据目录同步

```bash
# 在备库服务器执行
# 首次同步（完整同步）
rsync -avz --delete \
    kingbase@primary:/data/kingbase/data/ \
    /data/kingbase/data/

# 后续增量同步（支持断点续传）
rsync -avz --delete \
    --partial \
    --append \
    kingbase@primary:/data/kingbase/data/ \
    /data/kingbase/data/
```

## 五、常见问题

### 5.1 断点文件损坏

**问题**：`Invalid checkpoint file format`

**解决**：
```bash
# 备份损坏的断点文件
mv /data/kingbase/kfs/checkpoint/kfs_checkpoint.dat /data/kingbase/kfs/checkpoint/kfs_checkpoint.dat.bak

# 重新初始化断点
$KINGBASE_HOME/../kfs/bin/kfs_admin.sh checkpoint-reset
```

### 5.2 断点与实际数据不一致

**问题**：断点记录的文件与实际同步文件不匹配

**解决**：
```bash
# 清除断点，重新开始同步
$KINGBASE_HOME/../kfs/bin/kfs_admin.sh checkpoint-clear

# 或者使用校验模式
$KINGBASE_HOME/../kfs/bin/kfs_admin.sh verify-and-resume
```

### 5.3 断点续传后数据重复

**问题**：恢复后出现重复数据

**解决**：
```sql
-- 使用目标库的去重机制
-- 在 KDTS 配置中启用去重
"options": {
    "deduplication": true,
    "deduplication.key": "id"
}

-- 或手动删除重复数据
DELETE FROM table_name WHERE ctid NOT IN (
    SELECT MIN(ctid) 
    FROM table_name 
    GROUP BY id
);
```

### 6.2 定时保存断点脚本

```bash
#!/bin/bash
# checkpoint_backup.sh - 定时备份断点文件

CHECKPOINT_DIR="/data/kingbase/kfs/checkpoint"
BACKUP_DIR="/data/backup/checkpoint"
DATE=$(date +%Y%m%d)

mkdir -p $BACKUP_DIR

# 备份断点文件
if [ -f "$CHECKPOINT_DIR/kfs_checkpoint.dat" ]; then
    cp $CHECKPOINT_DIR/kfs_checkpoint.dat $BACKUP_DIR/kfs_checkpoint_$DATE.dat
    echo "断点文件已备份: $BACKUP_DIR/kfs_checkpoint_$DATE.dat"
fi

# 保留最近 30 天的备份
find $BACKUP_DIR -name "*.dat" -mtime +30 -delete
```

### 6.3 监控断点状态

```sql
-- KDTS 断点状态查询
SELECT 
    job_id,
    table_name,
    last_key,
    last_offset,
    status,
    update_time
FROM kdts_checkpoint
ORDER BY update_time DESC;

-- 检查长时间未更新的断点
SELECT * FROM kdts_checkpoint 
WHERE update_time < NOW() - INTERVAL '1 hour'
AND status = 'IN_PROGRESS';
```
