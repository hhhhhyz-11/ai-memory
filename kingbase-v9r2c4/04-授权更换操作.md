# KingbaseES V9R2C14 授权更换操作

> 本文档介绍 KingbaseES 许可证（License）的查看、更换和管理操作。

---

## 一、License 概述

### 1.1 License 类型

| 类型 | 说明 | 用途 |
|------|------|------|
| 评估 License | 有效期通常 30~90 天 | 测试评估 |
| 正式 License | 正式授权，有有效期限制 | 生产环境 |
| 永久 License | 无时间限制 | 永久授权用户 |
| 节点 License | 按 CPU 核数/服务器数量授权 | 企业版 |

### 1.2 License 文件

KingbaseES License 通常是一个加密的 license 文件：
- 文件名：`kingbase.key` 或 `syslicense.key`
- 存放位置：`$KINGBASE_HOME/license/` 或 `$KINGBASE_DATA/`

## 二、查看当前 License 信息

### 2.1 通过 SQL 查看

```sql
-- 连接数据库
ksql -U system -d test

-- 查看 License 信息
SELECT * FROM sys_license_info();

-- 或者
SELECT * FROM sys_license();

-- 查看 License 详细信息
SELECT 
    license_type,
    license_version,
    issue_date,
    expire_date,
    cpu_cores,
    license_nodes,
    license_users,
    is_valid,
    server_id
FROM sys_license_info();
```

### 2.2 通过命令行查看

```bash
# 查看 License 文件位置
ls -la $KINGBASE_HOME/license/
ls -la $KINGBASE_DATA/kingbase.key

# 查看 License 内容（加密格式，通常是二进制）
cat $KINGBASE_HOME/license/kingbase.key

# 尝试解析 License（如果有解密工具）
# $KINGBASE_HOME/bin/syslicense -info
```

### 2.3 License 信息字段说明

| 字段 | 说明 |
|------|------|
| `license_type` | License 类型（评估/正式/永久） |
| `issue_date` | 授权日期 |
| `expire_date` | 过期日期 |
| `cpu_cores` | 允许的 CPU 核心数 |
| `license_nodes` | 允许的节点数 |
| `license_users` | 允许的用户数 |
| `is_valid` | 是否有效 |
| `server_id` | 服务器 ID（硬件绑定） |

## 三、License 更换操作

### 3.1 获取新的 License 文件

1. 联系金仓销售或技术支持获取新 License
2. 获取 License 文件（`.key` 或 `.lic` 文件）
3. 确保新 License 与当前服务器硬件信息匹配

### 3.2 检查服务器硬件信息

```bash
# 查看服务器 IP（用于生成 license）
hostname -I
ip addr show

# 查看服务器序列号/硬件 ID
dmidecode -s system-serial-number
cat /sys/class/dmi/id/product_serial

# 查看 CPU ID
cat /proc/cpuinfo | grep "physical id" | head -1
cat /proc/cpuinfo | grep "processor" | wc -l

# 查看 MAC 地址
ip link show | grep ether
```

### 3.3 备份当前 License

```bash
# 备份当前 License 文件
su - kingbase

# 备份位置
mkdir -p $KINGBASE_HOME/license/backup_$(date +%Y%m%d)
cp $KINGBASE_HOME/license/kingbase.key $KINGBASE_HOME/license/backup_$(date +%Y%m%d)/
cp $KINGBASE_DATA/kingbase.key $KINGBASE_DATA/kingbase.key.bak.$(date +%Y%m%d)

# 或者直接备份整个 data 目录
tar -czpf /tmp/kingbase_data_backup_$(date +%Y%m%d).tar.gz $KINGBASE_DATA/
```

### 3.4 停止数据库

```bash
# 停止 KingbaseES 服务
sys_ctl -D $KINGBASE_DATA stop -m fast

# 确认进程已停止
ps -ef | grep kingbase | grep -v grep
```

### 3.5 替换 License 文件

```bash
su - kingbase

# 方法一：覆盖 kingbase.key
# 将新 license 文件上传到临时目录后复制
cp /tmp/new_license/kingbase.key $KINGBASE_HOME/license/kingbase.key
cp /tmp/new_license/kingbase.key $KINGBASE_DATA/kingbase.key

# 设置正确权限
chown kingbase:kingbase $KINGBASE_HOME/license/kingbase.key
chown kingbase:kingbase $KINGBASE_DATA/kingbase.key
chmod 644 $KINGBASE_HOME/license/kingbase.key
chmod 644 $KINGBASE_DATA/kingbase.key

# 方法二：如果 license 在其他目录，修改软链接或配置文件
# 检查 license 配置位置
grep -r "license" $KINGBASE_DATA/kingbase.conf
```

### 3.6 启动数据库

```bash
# 启动 KingbaseES
sys_ctl -D $KINGBASE_DATA start

# 等待启动完成
sleep 10

# 检查启动日志
tail -20 $KINGBASE_DATA/log/*.log
```

## 四、验证新 License

### 4.1 SQL 验证

```sql
-- 连接数据库
ksql -U system -d test

-- 查看 License 状态
SELECT * FROM sys_license_info();

-- 检查是否生效
SELECT 
    license_type,
    expire_date,
    is_valid,
    server_id
FROM sys_license_info();

-- 查看 License 有效期
SELECT 
    expire_date - current_date AS days_until_expiry
FROM sys_license_info();
```

### 4.2 常用验证 SQL

```sql
-- 检查 License 类型
SELECT license_type FROM sys_license_info();

-- 检查过期时间
SELECT 
    expire_date,
    CASE 
        WHEN expire_date < current_date THEN '已过期'
        WHEN expire_date < current_date + interval '30 days' THEN '30天内过期'
        ELSE '有效'
    END AS status
FROM sys_license_info();

-- 检查授权节点数
SELECT license_nodes FROM sys_license_info();

-- 检查授权核心数
SELECT cpu_cores FROM sys_license_info();

-- 检查授权用户数
SELECT license_users FROM sys_license_info();
```

### 4.3 常见问题排查

```sql
-- 如果 License 无效，可能的报错：
-- FATAL: license error: license expired
-- FATAL: license error: invalid license
-- FATAL: license error: hardware mismatch

-- 查看详细错误
SELECT * FROM sys_license_info() \x
```

## 五、License 过期处理

### 5.1 过期前准备

```bash
# 脚本：检查 License 过期时间
#!/bin/bash
# check_license.sh

RESULT=$(ksql -U system -d test -t -c "SELECT expire_date::text FROM sys_license_info();")
EXPIRE_DATE=$(echo $RESULT | tr -d ' ')

echo "License 过期日期: $EXPIRE_DATE"

# 计算剩余天数
DAYS_LEFT=$(( ($(date -d "$EXPIRE_DATE" +%s) - $(date +%s)) / 86400 ))
echo "剩余天数: $DAYS_LEFT 天"

if [ $DAYS_LEFT -lt 30 ]; then
    echo "⚠️  License 将在 30 天内过期，请及时更换！"
fi
```

### 5.2 已过期处理

如果 License 已过期：

1. **紧急处理**：联系金仓技术支持申请临时 License
2. **生产环境**：提前准备备机，择机更换 License
3. **测试环境**：申请评估 License 延期

```bash
# 查看过期 License 的具体错误
tail -100 $KINGBASE_DATA/log/*.log | grep -i license
```

## 六、License 管理最佳实践

### 6.1 License 管理脚本

```bash
#!/bin/bash
# manage_license.sh - License 管理脚本

KINGBASE_HOME=/opt/KingbaseES/V9R2C14
KINGBASE_DATA=/data/kingbase/data
LICENSE_BACKUP=/opt/kingbase_license_backup
TEMP_LICENSE=/tmp/new_license

mkdir -p $LICENSE_BACKUP

usage() {
    echo "用法: $0 {check|backup|install|status}"
    echo "  check   - 查看当前 License 信息"
    echo "  backup  - 备份当前 License"
    echo "  install - 安装新 License（需要指定文件路径）"
    echo "  status  - 查看 License 状态"
    exit 1
}

check_license() {
    echo "=== 当前 License 信息 ==="
    ksql -U system -d test -c "SELECT * FROM sys_license_info();" 2>/dev/null || echo "无法连接到数据库"
}

backup_license() {
    DATE=$(date +%Y%m%d)
    echo "备份 License 到 $LICENSE_BACKUP/license_$DATE.key"
    if [ -f "$KINGBASE_DATA/kingbase.key" ]; then
        cp $KINGBASE_DATA/kingbase.key $LICENSE_BACKUP/license_$DATE.key
        echo "备份完成"
    else
        echo "未找到 License 文件"
    fi
}

install_license() {
    if [ -z "$2" ]; then
        echo "请指定 License 文件路径"
        exit 1
    fi
    
    LICENSE_FILE=$2
    if [ ! -f "$LICENSE_FILE" ]; then
        echo "License 文件不存在: $LICENSE_FILE"
        exit 1
    fi
    
    echo "安装新 License..."
    
    # 停止数据库
    sys_ctl -D $KINGBASE_DATA stop -m fast
    
    # 备份旧 License
    backup_license
    
    # 复制新 License
    cp $LICENSE_FILE $KINGBASE_DATA/kingbase.key
    cp $LICENSE_FILE $KINGBASE_HOME/license/kingbase.key
    chown kingbase:kingbase $KINGBASE_DATA/kingbase.key
    chown kingbase:kingbase $KINGBASE_HOME/license/kingbase.key
    
    # 启动数据库
    sys_ctl -D $KINGBASE_DATA start
    
    sleep 5
    
    # 验证
    check_license
}

case "$1" in
    check) check_license ;;
    backup) backup_license ;;
    install) install_license $@ ;;
    status) check_license ;;
    *) usage ;;
esac
```

### 6.2 定时检查任务

```bash
# 添加 crontab 定时检查 License
crontab -e

# 每天早上 9 点检查 License 状态
0 9 * * * /opt/scripts/check_license.sh >> /var/log/license_check.log 2>&1

# License 过期前 30 天提醒
```

### 6.3 License 续期流程图

```
1. 联系金仓技术支持 → 获取 License 申请表格
        ↓
2. 填写服务器信息（IP、CPU、MAC 地址）
        ↓
3. 提交申请 → 获取新 License 文件
        ↓
4. 测试环境验证
        ↓
5. 生产环境安装（参考 3.5 节步骤）
        ↓
6. 验证新 License 生效
        ↓
7. 完成
```

## 七、常见问题

### 7.1 License 文件不存在

**问题**：`FATAL: could not open license file`

**解决**：
```bash
# 确认 License 文件位置
find / -name "kingbase.key" -o -name "*.key" 2>/dev/null | grep -i kingbase

# 重新复制到正确位置
cp /path/to/kingbase.key $KINGBASE_DATA/kingbase.key
```

### 7.2 硬件信息不匹配

**问题**：`FATAL: license error: hardware mismatch`

**解决**：
1. 确认服务器硬件是否更换（MAC 地址、CPU 等）
2. 联系金仓技术支持重新生成匹配新硬件的 License

### 7.3 License 已过期

**问题**：`FATAL: license error: license expired`

**解决**：
1. 联系金仓销售申请续期
2. 获取新的 License 文件后按步骤安装
