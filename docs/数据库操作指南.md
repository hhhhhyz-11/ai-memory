## 一、查看监听

### 1.1 查看数据库端口监听状态

```bash
# 查看指定端口是否在监听
netstat -tlnp | grep 54321

# 查看所有监听端口
netstat -tlnp

# 查看端口占用情况
lsof -i:54321
```

**输出示例：**
```
tcp        0      0 0.0.0.0:54321           0.0.0.0:*               LISTEN      12345/kingbase
```

### 1.2 查看数据库连接

```bash
# 查看当前连接数
/home/kingbase/Server/bin/ksql -U system -d test -c "SELECT count(*) FROM sys_stat_activity;"

# 查看详细连接信息
/home/kingbase/Server/bin/ksql -U system -d test -c "SELECT * FROM sys_stat_activity;"
```

### 1.3 查看数据库服务状态

```bash
# KingBase 服务状态
/home/kingbase/Server/bin/sys_ctl -D /home/kingbase/data status

# systemd 服务方式
systemctl status kingbase-master
```

## 二、修改 SQL 脚本权限

### 2.1 常用权限设置

```bash
# 添加执行权限
chmod +x /path/to/script.sql

# 755 权限（所有者读写执行，其他用户读执行）
chmod 755 /path/to/script.sql

# 644 权限（所有者读写，其他用户只读）
chmod 644 /path/to/script.sql

# 递归修改目录下所有 .sql 文件
chmod +x /path/to/*.sql
```

### 2.2 文件所有者修改

```bash
# 修改文件所有者
chown kingbase:kingbase /path/to/script.sql

# 递归修改目录所有者
chown -R kingbase:kingbase /path/to/scripts/
```

### 2.3 查看文件权限

```bash
# 查看详细权限
ls -la /path/to/script.sql

# 查看目录权限
ls -ld /path/to/scripts/
```

## 三、还原数据库操作

### 3.1 准备工作

```bash
# 1. 确认备份文件存在
ls -la /path/to/backup.sql

# 2. 确认数据库服务运行中
/home/kingbase/Server/bin/sys_ctl -D /home/kingbase/data status

# 3. 创建目标数据库（如果需要）
/home/kingbase/Server/bin/ksql -U system -d postgres -c "CREATE DATABASE target_db;"
```

### 3.2 还原数据库（sys_dump 导出文件）

```bash
# 还原整个数据库
/home/kingbase/Server/bin/ksql -U system -d target_db -f /path/to/backup.sql

# 还原并显示执行过程
/home/kingbase/Server/bin/ksql -U system -d target_db -f /path/to/backup.sql -v ON_ERROR_STOP=1

# 指定编码还原
/home/kingbase/Server/bin/ksql -U system -d target_db --encoding=UTF8 -f /path/to/backup.sql
```

### 3.3 交互式还原

```bash
# 进入数据库后执行
/home/kingbase/Server/bin/ksql -U system -d target_db

# 在数据库内执行
\i /path/to/backup.sql
```

### 3.4 常见还原问题处理

| 问题 | 解决方案 |
|------|----------|
| 目标数据库不存在 | 先创建数据库 `CREATE DATABASE target_db;` |
| 权限不足 | 使用 system 用户或授权 |
| 字符集不匹配 | 添加 `--encoding=UTF8` 参数 |
| 表已存在 | 先清空或删除 `DROP DATABASE;` 重建 |
| 还原中断 | 使用 `-v ON_ERROR_STOP=1` 立即停止并报错 |

## 四、附录

### 4.1 sys_dump 备份导出（供参考）

```bash
# 导出整个数据库
./sys_dump -U system -W -d test -f test_backup.sql

# 导出指定表
./sys_dump -U system -W -d test -t my_schema.my_table -f table_backup.sql

# 只导出表结构
./sys_dump -U system -W -d test -s -f structure.sql

# 只导出数据
./sys_dump -U system -W -d test -a -f data.sql
```
