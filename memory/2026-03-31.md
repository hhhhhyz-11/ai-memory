## PostgreSQL 15 Docker 容器化迁移（完整记录）

### 迁移背景
- 源环境：物理服务器 PostgreSQL 15（Systemd 管理），node1（192.168.0.50）
- 目标环境：Docker 容器 PostgreSQL 15.15，node2（192.168.0.53）
- 宿主机挂载路径：/home/postgres15/data
- 迁移内容：19 个业务数据库 + 所有角色权限 + PostGIS 空间扩展

### 迁移步骤

**第一阶段：源端数据导出（node1）**
```bash
systemctl stop postgresql-15
sudo -u postgres /usr/pgsql-15/bin/pg_dumpall > /home/postgres_full_dump.sql
scp /home/postgres_full_dump.sql root@node2:/home/
```

**第二阶段：目标端构建（node2）**
- 自定义 Dockerfile（解决 zh_CN.UTF-8 和 PostGIS 缺失）
- docker-compose.yml 配置

**第三阶段：干净导入**
```bash
mkdir -p /home/postgres15/data
chown -R 999:999 /home/postgres15/data
rm -rf /home/postgres15/data/*  # 彻底清理残留
docker-compose up -d --build
docker exec -i postgres15 psql -U postgres -f /tmp/postgres_full_dump.sql
docker exec -it postgres15 psql -U postgres -c "ALTER USER postgres WITH PASSWORD 'Yst@163.com';"
```

### 核心避坑点
1. Locale 缺失 → Dockerfile 安装 locales + localedef 生成 zh_CN
2. PostGIS 缺失 → apt-get 安装 postgresql-15-postgis-3
3. 数据残留 → rm -rf data/* 彻底清理
4. 权限冲突 → chown -R 999:999
5. 密码覆盖 → 导入后手动 ALTER USER postgres WITH PASSWORD

### 验证命令
```bash
docker exec -it postgres15 psql -U postgres -c "\l"  # 查看所有库
docker exec -it postgres15 psql -U postgres -c "SELECT postgis_full_version();"  # PostGIS版本
```
