### sessions.json 堆积问题调查与修复

**问题现象：** Web UI 显示大量 session（最高达 2558 个文件），sessions.json 注册数 234 条，其中 dreaming session 140 个

**根因定位：**
- memory-core dreaming 功能超时后不停重试，每次创建 3 个 session（light/REM/deep）
- 连续失败 13 次后 session 堆积成灾
- sessions.json 无 maxEntries 限制、无过期删除机制、无自动清理
- session-corpus 和 short-term-recall.json 恶性膨胀（75206 行、2.5MB），导致处理时间超过 600s timeout

**修复操作：**
1. ✅ 禁用 dreaming：`plugins.entries.memory-core.config.dreaming.enabled` → `false`
2. ✅ 清理 140 个 dreaming session（从 sessions.json）
3. ✅ 删除 772 个 orphaned 文件（jsonl/trajectory/lock）
4. ✅ 备份并清理 `.dreams/` 目录（4.6MB → 372KB）
   - short-term-recall.json 重置
   - events.jsonl 重置
   - session-corpus 保留近 6 天，归档 14 个旧文件
5. ✅ 重新开启 dreaming（数据清理后）

**新增配置（防止再次膨胀）：**
```json
"session": {
  "maintenance": {
    "mode": "enforce",
    "maxEntries": 200,
    "pruneAfter": "30d",
    "rotateBytes": "50mb"
  }
}
```
