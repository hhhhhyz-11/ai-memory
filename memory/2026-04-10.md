### 升级过程（kpatch 原位升级）

**踩坑记录：**

1. **kpatch 路径不能包含 KESRealPro**
   - 错误：`./kpatch -k /home/kingbase/KESRealPro/V009R002C013/Server ...`
   - 报错：`kingbase_path contains:KESRealPro, should not use with:-k *KESRealPro*`
   - 解决：改用软链接路径 `-k /home/kingbase/Server`

2. **Kingbase 必须先启动才能执行 kpatch check**
   - kpatch 会检查 kingbase 状态，如果服务未启动会报错：`kingbase not running`
   - 解决：先 `sys_ctl -D /home/kingbase/data start`

3. **PID 文件残留导致启动失败**
   - 报错：`lock file "kingbase.pid" already exists`
   - 解决：`sys_ctl -D /home/kingbase/data stop` 清理后再启动

**正确升级步骤：**

```bash
# 1. 备份数据
mkdir -p /home/kingbase/data_bak
cp -r /home/kingbase/data/ /home/kingbase/data_bak/

# 2. 上传补丁包到 /home/kingbase/Kpatch/patch_packages/

# 3. 解压 kpatch
cd /home/kingbase/Kpatch/patch_packages
tar -zxf kingbase-server-V009R002C013PS014-linux-x86_64.tar \
  --strip-components 2 -C .. ./bin/kpatch

# 4. 确保数据库运行中
cd /home/kingbase/KESRealPro/V009R002C013/Server/bin
./sys_ctl -D /home/kingbase/data start
./ksql -d test -U system  # 验证连接

# 5. 执行 kpatch 升级（注意 -k 路径用软链接，不用 KESRealPro 路径）
cd /home/kingbase/Kpatch
./kpatch -k /home/kingbase/Server \
  -D /home/kingbase/data \
  -t V009R002C013PS014 apply single
# 输入 y 确认

# 6. 验证升级结果
cd /home/kingbase/Server/bin
./kingbase -V
./ksql -V
```
