# 2026-03-18 工作日志

## 今日完成

### 1. Nginx 转发配置问题排查
- **问题**：测试域名 `oa.changjiafs.com` 转发到生产 `oa.gdyunst.com` 时，返回 404
- **原因**：`proxy_pass` 尾部斜杠问题
  - `proxy_pass https://oa.gdyunst.com/oaweb/` 会把 `/auth/oactual 变成 `/oaweb/auth/oactual`
- **解决**：去掉 `/oaweb/` 路径
- **配置层级**：
  - 测试机 → 生产机 → 第二层转发 → 后端 `10.110.81.137:8088`

### 2. GitLab CI 自动合并 MR
- **需求**：开发无合并权限，MR 测试通过后自动合并
- **方案**：在 CI pipeline 末尾添加 `auto-merge` job
- **实现**：
  - 用有 Merge 权限的 PAT（Personal Access Token）
  - 调用 GitLab Merge API 自动合并
  - 检查 MR 状态：`opened` + `can_be_merged`
- **配置**：
  - 添加 `MERGE_TOKEN` 变量
  - 使用 Alpine 镜像执行 curl + jq

## 学习总结

- Nginx `proxy_pass` 尾部斜杠的行为：
  - 有斜杠：Nginx 会替换 location 匹配的路径
  - 无斜杠：保留原始请求路径
- GitLab API 合并 MR：`PUT /projects/:id/merge_requests/:iid/merge`
