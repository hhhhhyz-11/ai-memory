# 学习总结 2026-04-08

## KingBase V9R2C4 主备 / 读写分离集群部署

---

### 1. install.conf 配置核心参数

关键配置项（决定集群形态）：

```properties
# 节点列表，首个即为主节点
all_ip=(192.168.56.11 192.168.56.12)

# 集群安装路径（需不存在）
install_dir="/home/kingbase/cluster/oracle"

# VIP，用于故障漂移
virtual_ip="192.168.56.15/24"

# 必须与 VIP 同网段网卡
net_device=(eth1 eth1)
net_device_ip=(192.168.56.11 192.168.56.12)

# 信任网关（需可达），测试环境可设为空或本地网关IP
trusted_servers="192.168.56.1"

# SSH直连模式（不依赖securecmdd则为1）
deploy_by_sshd=0
```

> `all_ip` + `virtual_ip` 是主备集群和读写分离集群共用的核心配置，配置了就是 RWC 集群。

---

### 2. 目录结构踩坑

**错误原因：**
```
install_dir="/home/kingbase/cluster/kingbase"
# 但 bin/include/lib/share 直接在 kingbase/ 下，未在 Server/ 子目录
```

**KingBase 期望结构：**
```
${install_dir}/
└── Server/
    ├── bin/
    ├── include/
    ├── lib/
    └── share/
```

**正确做法：先创建 Server 目录再解压 db.zip：**
```bash
mkdir -p /home/kingbase/cluster/kingbase/Server
cd /home/kingbase/cluster/kingbase/Server
unzip db.zip
```

---

### 3. 防火墙导致集群部署失败

**症状：** `Failed to ping trusted_servers`

**解决（测试环境）：**
```bash
systemctl stop firewalld
systemctl disable firewalld
```

---

### 4. SSH 免密脚本 trust_cluster.sh

- 由 KingBase 官方提供：`${install_dir}/ClientTools/guitools/DeployTools/zip/`
- 需要 root 执行，配置 kingbase 用户和 root 之间的双向免密
- 脚本自动处理 kingbase↔kingbase、root↔root、kingbase↔root 的互信

---

### 5. 主备 vs 读写分离 — 核心区别

| 对比项 | 主备集群 | 读写分离集群 |
|--------|---------|-------------|
| 目的 | HA 切换 | 读写分离 + HA |
| 备机作用 | 热备，不承接业务 | 主动承接读流量 |
| 收益 | 可用性 | 读性能线性扩展 + 可用性 |

**结论：install.conf 配置了 `all_ip` + `virtual_ip` 的集群，本质上就是读写分离集群（RWC）。** 主备集群只是 RWC 的一个子集——备机不承接读流量而已。

---

### 6. 应用层读写分离实现

KingBase 流复制是底层基础，应用层两种方案：

- **方案 A（推荐，改动小）：** 应用层直连，写→VIP，读→备机
- **方案 B：** HAProxy + Keepalived 对备机做负载均衡，对应用暴露单一读地址

备机只读验证：
```sql
-- 备库写入应报错
INSERT INTO t1 VALUES (1);
-- 报错：cannot execute INSERT in a read-only transaction
```
