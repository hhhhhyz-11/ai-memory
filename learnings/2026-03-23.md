# 2026-03-23 学习总结

## 📚 技术知识

### GitLab CI/CD + SonarQube 完整流水线

1. **SonarQube 扫描配置**
   - 增量代码扫描（增量分析）
   - 质量门（Quality Gate）检查
   - 通过 API 获取新增问题详情：Bug、漏洞、代码异味

2. **PDF 报告生成**
   - 使用 Python + WeasyPrint 生成中文 PDF
   - 报告包含：文件名、行号、严重性、问题描述
   - 按问题类型分类显示

3. **MinIO 对象存储**
   - MinIO Client (mc) 命令行工具
   - `mc alias set` 配置连接
   - `mc mb` 创建 bucket
   - `mc anonymous set download` 设置公开下载权限
   - **重要**：9001 是 Console 端口，9000 是 API 端口

4. **Docker 多阶段构建**
   - Dockerfile 可同时包含 Maven JDK 和 Python 环境
   - 需安装中文字体支持（fonts-wqy-zenhei）

5. **MR 评论与通知**
   - 使用 GitLab API 发送 MR 评论
   - 钉钉 Webhook 机器人发送通知

### GitHub 仓库管理
- 创建项目仓库并上传流水线配置
- GitLab 和 GitHub 仓库协同管理

## ⚠️ 踩坑记录

1. **MinIO 端口错误**
   - 问题：使用 9001 端口上传失败
   - 原因：9001 是 Console 控制台端口，API 端口是 9000

2. **Bucket 权限问题**
   - 问题：上传的 PDF 无法公开访问
   - 解决：使用 `mc anonymous set download` 设置公开访问或配置 policy

## 💡 经验教训

- MinIO 有两个端口：9000 (API) 和 9001 (Console)，配置时需注意区分
- 对象存储 bucket 需要显式设置公开访问权限
- 完整的 CI/CD 流水线需要整合代码扫描、报告生成、存储、通知多个环节
- 中文 PDF 生成需要字体支持
