# 学习总结 2026-04-09

## KingBase 主备集群读写分离

- **主备集群**：备机平时不承接业务，只做 HA 切换，主故障时接管
- **读写分离集群**：备机主动承接读流量，提升整体吞吐，架构相同但配置更完整
- 区分关键参数：`virtual_ip` + `all_ip` → 读写分离集群；只有流复制 → 主备
- 应用层路由：写 → VIP（主），读 → 备机 IP 或 HAProxy

## KingBase 版本升级（V009R002C013PS014）

- **补丁包命名规则**：`kingbase-server-<版本号>-linux-<CPU>.tar`
- **kpatch 升级**：不需要 sys_dump/sys_restore，原位打补丁，停机时间短
- **升级路径**：必须匹配基线版本（PS014 仅支持 PS007 / PS009 基线）
- **坑**：V9R4C16 ≠ V9R2C13，不是同一个系列，无法通用补丁包
- **kpatch 解压**：`--strip-components 2 -C .. ./bin/kpatch` → 解压到上级目录的 bin 子目录

## tcpdump HTTP CONNECT 抓包分析

- HTTP CONNECT 方法：建立正向代理隧道，用于访问 HTTPS 网站
- `CONNECT rz.weijing.gov.cn:443` + `200 Connection established` = 隧道建立成功
- 后续流量是加密的 HTTPS（tcpdump 看不到内容），但能看到大量 HTTP 分片 ACK

## Vagrant 多节点管理

- `vagrant up <name>` 只启动指定节点，不影响其他节点
- Windows Vagrantfile 路径：`F:\virtualbos\Vagrantfile`
- WSL 和 Windows 的 Vagrantfile 需要同步更新
