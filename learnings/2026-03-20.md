# 2026-03-20 学习总结

## 📚 技术知识

### GitLab CI + SonarQube 配置
- **自动合并 MR**：使用 `when: manual` 会阻止流水线自动合并，需去掉手动触发
- **合并 API 405 错误**：合并请求返回 405 时，尝试添加 `"squash": false` 参数
- **interruptible 参数**：GitLab 11.1.4 不支持 `interruptible` 键，需移除

### 流水线优化
- **重复流水线问题**：提交代码时旧 pipeline 还在运行，新 pipeline 等待
- **解决方案**：在 job 脚本中使用 GitLab API 取消同一分支的旧 pipeline
- **API 调用**：`POST /projects/:id/pipelines/:pipeline_id/cancel`

## ⚠️ 踩坑记录

1. `when: manual` 导致流水线不会自动合并
2. MR 状态不允许合并或 squash 配置导致 405 错误
3. GitLab 版本不支持 `interruptible` 参数
4. 提交代码时旧 pipeline 与新 pipeline 冲突

## 💡 经验教训

- 旧版 GitLab 不支持某些现代 CI 参数，配置时需注意版本兼容性
- 流水线优化可考虑自动取消旧运行，避免资源浪费和等待
- 合并 MR 前需确保 MR 状态允许合并（无冲突、审批通过等）
