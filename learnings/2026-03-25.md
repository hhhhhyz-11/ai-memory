# 2026-03-25 学习总结

## 📚 技术知识

### Redis 集群水平扩容

1. **添加新节点步骤**
   - 在新服务器创建 Redis 数据目录（如 /usr/local/redis-cluster/36379）
   - 复制并修改配置文件（端口、cluster-config-file）
   - 启动 Redis 实例

2. **添加主节点到集群**
   ```bash
   redis-cli -a <password> --cluster add-node <新主节点IP:端口> <集群任一节点IP:端口>
   ```

3. **Hash 槽迁移（重要）**
   - 使用 `reshard` 命令分配槽
   - 需要先从集群其他节点迁移槽到新主节点
   - 交互式：`redis-cli -a <password> --cluster reshard <节点>`
   - 输入要迁移的槽数量、目标节点ID、源节点ID（可用 all）

4. **添加从节点**
   ```bash
   redis-cli -a <password> --cluster add-node <新从节点IP:端口> <集群任一节点IP:端口>
   # 关联到主节点
   redis-cli -a <password> -c -h <从节点> -p <端口>
   > cluster replicate <主节点ID>
   ```

5. **下线节点**
   - 必须先将目标节点的 hash 槽迁移到其他节点
   - 使用 `reshard` 将槽迁出
   - 然后删除节点：`redis-cli -a <password> --cluster del-node <节点> <节点ID>`

### NFS 挂载管理

- 失效文件句柄处理
- `umount -l` 懒卸载处理 busy 状态的挂载点

## ⚠️ 踩坑记录

1. **Redis 连接报错 I/O error**
   - 可能原因：端口未开放、防火墙、Redis 进程未启动
   - 排查：telnet/nc 测试端口连通性、ps 检查进程

2. **下线节点无需手动释放 hash 槽**
   - 通过 reshard 将槽迁移走后，集群会自动处理
   - 直接删除节点即可

## 💡 经验教训

- Redis 集群添加节点必须先添加主节点，再分配槽，最后添加从节点
- 下线节点前必须先迁移走其承担的 hash 槽，否则数据会丢失
- reshard 时用 all 作为源节点可以将集群中所有主节点的槽均匀分配到目标节点
- NFS 挂载点失效时使用 `umount -l` 懒卸载
