## 📚 技术知识

### 1. GitLab 版本升级原则
- **必须遵循官方升级路径**，不能跨版本升级
- 大版本升级需要逐级递增（如 11.x → 12.x → 13.x...）
- 11.1.4 → 16.1.8 需要经过 17 个中间版本

### 2. Docker GitLab 环境配置
- 三个核心卷：`/var/opt/gitlab`（数据）、`/var/log/gitlab`（日志）、`/etc/gitlab`（配置）
- 端口映射：80 → 8080, 443 → 4443, 22 → 2222
- 语言环境 `en_US.UTF-8` 必须匹配，否则 PostgreSQL 启动失败

### 3. GitLab 备份恢复
```bash
# 创建备份
gitlab-rake gitlab:backup:create

# 恢复备份
gitlab-rake gitlab:backup:restore BACKUP=<backup_timestamp>
```

## ⚠️ 踩坑记录

### 踩坑 1: gitlab-secrets.json 导致 CI/CD 500 错误
- **问题**：旧版本配置文件导致所有项目 CI/CD 页面 500 错误（`aes256_gcm_decrypt`）
- **原因**：加密密钥不兼容
- **解决方案**：
  - Rails 控制台删除所有变量：`project.variables.destroy_all`
  - 数据库彻底清除：`DELETE FROM ci_variables; DELETE FROM ci_group_variables;`
  - 清除 tokens 和加密字段

### 踩坑 2: GitLab 克隆地址变成容器 ID
- **问题**：升级后克隆地址变成 `git@7bd63ee2fcc6:inner/data-board.git`（容器ID）而非 IP 地址
- **原因**：旧配置文件 external_url 未更新
- **解决方案**：修改 `/etc/gitlab/gitlab.rb` 中的 `external_url` 为正确地址

### 踩坑 3: 语言环境不匹配
- **问题**：PostgreSQL 无法启动
- **原因**：容器语言环境与主机不一致
- **解决方案**：确保主机和容器都使用 `en_US.UTF-8`

## 💡 经验教训

1. **升级前务必先恢复到官方版本再升级**，不能从第三方修改版直接升级
2. **每次升级后至少验证 2 小时**再继续下一步
3. **做好完整备份**：物理备份（etc/data/log）+ 容器内备份
4. **准备好回滚方案**：删除容器后可立即用旧镜像重启
5. **清除 CI/CD 变量是解决加密密钥问题的最终方案**
