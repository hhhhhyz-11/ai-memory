## 🧠 技术知识

### 1. SonarQube 增量代码扫描

**核心参数：**
- `-Dsonar.newCode.referenceBranch=uat` — 增量对比分支
- `-Dsonar.pullrequest.key` — PR 模式

**适用场景：**
- feature 分支对比 uat 分支做增量扫描
- 只检测新增/修改的代码，不扫全量

**GitLab CI 配置示例：**
```yaml
sonarqube-check:
  stage: sonar
  script:
    - maven sonar:sonar \
      -Dsonar.host.url=$SONAR_URL \
      -Dsonar.token=$SONAR_TOKEN \
      -Dsonar.newCode.referenceBranch=uat \
      -Dsonar.projectKey=your-project
```

### 2. SonarQube + GitLab 门禁配置

**需求：** master/uat/release 分支合并时必须通过质量门禁

**配置步骤：**
1. **SonarQube 侧：** 设置质量门禁（Quality Gate）
2. **GitLab 侧：**
   - 进入 Settings → Protected branches
   - 设置 master/uat/release 为保护分支
   - 开启 "Require status checks to pass before merging"
   - 添加 SonarQube 检查

**分支关系继承：**
- SonarQube 自动识别 GitLab 分支关系
- 无需手动配置 branch → PR 的映射

### 3. AI 短剧制作工具链

**推荐工具组合：**
| 环节 | 工具 | 用途 |
|------|------|------|
| 图片生成 | Leonardo.ai | AI 生图 |

**工作流程：**

4. 用 CapCut 剪辑合成

## 📝 经验教训

1. **SonarQube 增量扫描只适用于同分支对比**，PR 模式需要配置 `sonar.pullrequest.*` 参数
2. **GitLab 保护分支门禁** 需要先在 SonarQube 设置质量门禁，否则 GitLab 检测不到状态
3. **AI 短剧制作** 建议先做分镜脚本，再生成素材，避免返工
