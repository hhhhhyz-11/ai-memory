# Learnings 2026-04-15

## 技术知识

### 1. Kingbase kpatch 原位升级
- kpatch 路径不能包含 KESRealPro，否则报错：`kingbase_path contains:KESRealPro, should not use with:-k *KESRealPro*`
- 解决：使用软链接路径 `-k /home/kingbase/Server`
- Kingbase 必须先启动才能执行 kpatch check，kpatch 会检查 kingbase 状态
- PID 文件残留会导致启动失败，报错：`lock file "kingbase.pid" already exists`

### 2. sys_dump 备份导出
- 导出整个数据库：`./sys_dump -U system -W -d test -f test_backup.sql`
- 导出指定表：`./sys_dump -U system -W -d test -t my_schema.my_table -f table_backup.sql`
- 只导出表结构：`./sys_dump -U system -W -d test -s -f structure.sql`
- 只导出数据：`./sys_dump -U system -W -d test -a -f data.sql`

### 3. OpenClaw Browser
- 当前版本 `browser` 子命令不支持 `install` 参数
- 浏览器自动化使用 H 节点 Chrome 解决

### 4. 工作区整理规范
- 业务配置文件按项目归类到独立目录
- 流水线文件、操作说明等分类存放
- 系统文件（MEMORY/SOUL/USER/AGENTS）保留在根目录
