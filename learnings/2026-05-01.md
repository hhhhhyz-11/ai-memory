## 📚 技术知识

### OpenClaw Session 维护配置
新增 `session.maintenance` 配置防止 session 堆积膨胀：
```json
"session": {
  "maintenance": {
    "mode": "enforce",
    "maxEntries": 200,
    "pruneAfter": "30d",
    "rotateBytes": "50mb"
  }
}
```
- `maxEntries=200`：sessions.json 最多保留 200 条，超出后自动清理
- `pruneAfter=30d`：30 天未使用的 session 标记为待删除
- `rotateBytes=50mb`：单个 session corpus 超过 50MB 时轮转

## 💡 经验教训

- **OpenClaw dreaming** 超时后会自动重试，每次创建 3 个 session（light/REM/deep），连续失败会导致 session 爆发式增长
- 修复思路：禁用 dreaming → 批量清理 session → 修复根因 → 重新开启
- 配置文件操作流程：备份 → 修改 → `openclaw doctor --fix` → 汇报状态
