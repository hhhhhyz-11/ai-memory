# 2026-04-13 学习总结

## 技术知识

### KingBase kpatch 原位升级

- **升级原理**：kpatch 在原目录进行热补丁升级，无需重建数据库
- **版本路径**：使用软链接路径（`-k /home/kingbase/Server`），而非 KESRealPro 路径
- **升级后目录结构**：
  - `Server` 软链接自动切换到新版本
  - 旧版本备份为 `Server_<timestamp>`
  - 数据目录 `data` 保持不变

### KingBase 常用命令

| 命令 | 用途 |
|------|------|
| `sys_ctl -D <data> start` | 启动数据库 |
| `sys_ctl -D <data> stop` | 停止数据库 |
| `ksql -d <db> -U <user>` | 连接数据库 |
| `kingbase -V` | 查看版本 |
| `sys_dump` | 备份工具 |
| `sys_restore` | 恢复工具 |

## 踩坑记录

### kpatch 路径包含 KESRealPro 导致失败

- **错误信息**：`kingbase_path contains:KESRealPro, should not use with:-k *KESRealPro*`
- **原因**：kpatch 检测到路径包含 KESRealPro 关键字会报错
- **解决**：使用软链接路径 `-k /home/kingbase/Server`

### Kingbase 未启动导致 kpatch check 失败

- **错误信息**：`kingbase not running`
- **原因**：kpatch 执行前会检查数据库运行状态
- **解决**：先执行 `sys_ctl -D /home/kingbase/data start`

### PID 文件残留导致启动失败

- **错误信息**：`lock file "kingbase.pid" already exists`
- **原因**：异常停止后 PID 文件未清理
- **解决**：先 `sys_ctl stop` 清理，再重新启动

## 经验教训

1. **升级前务必备份数据**：`cp -r /home/kingbase/data /home/kingbase/data_bak/`
2. **升级前验证数据库可连接**：`ksql -d test -U system`
3. **kpatch 升级耗时短**（约23秒），但需预留回滚时间
4. **升级后验证**：确认 `kingbase -V` 显示新版本号
