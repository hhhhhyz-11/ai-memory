# 学习总结 2026-03-26

## MySQL Docker 迁移要点

### 密码重置（MySQL 8）

MySQL 8 的变化：
- 不支持 `PASSWORD()` 函数
- 默认使用 `caching_sha2_password` 插件
- `skip-grant-tables` 模式下无法执行 CREATE USER / GRANT，需要直接操作 `mysql.user` 表

正确流程：
1. 开启 `skip-grant-tables`
2. 用 INSERT 直接写入用户表（需指定所有 NOT NULL 字段）
3. plugin 字段设为 `mysql_native_password`
4. 重启 MySQL（去掉 skip-grant-tables）后正常登录

### Docker MySQL 权限

Docker 镜像内 MySQL 运行用户是 mysql (UID 999)，数据目录挂载时需要：
```bash
chown -R 999:999 /home/mysql/data
```

### 备份参数说明

```bash
--single-transaction  # InnoDB 表在事务中备份，不锁表
--triggers            # 备份触发器
--routines           # 备份存储过程和函数
--events             # 备份事件调度器
--set-gtid-purged=OFF # 关闭 GTID purge，避免导入时权限问题
```

## OpenClaw 配置

模型配置在 `models.providers.minimax-cn.models` 数组中，切换默认模型需要同时修改：
- `models.providers.minimax-cn.models`（添加新模型）
- `agents.defaults.model.primary`（设为主流）
- `agents.defaults.models`（添加别名映射）
