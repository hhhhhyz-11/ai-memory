## 技术知识

### MySQL 8 用户认证
- MySQL 8 不再推荐使用 `PASSWORD()` 函数，建议使用明文密码
- `root@192.168.%` 和 `root@localhost` 是两个独立的用户账户
- Docker MySQL 在 `skip-grant-tables` 模式下可以通过直接操作 `mysql.user` 表来创建用户
- 使用 `mysql_native_password` 插件进行认证

### RabbitMQ 数据迁移
- 从 Systemd 管理迁移到 Docker 部署
- 需要注意配置文件路径和数据目录的映射

### OpenClaw 模型切换
- 支持运行时切换全局模型
- 修改配置文件后自动生效

## 踩坑记录

### MySQL 密码重置失败
- **问题**：MySQL 8 执行 `SET PASSWORD` 报错
- **原因**：MySQL 8 已废弃 `PASSWORD()` 函数
- **解决**：使用 `ALTER USER` 或直接插入 `mysql.user` 表（skip-grant-tables 模式下）

### Docker MySQL 用户创建
- **问题**：无法通过常规方式创建 root 用户
- **原因**：Docker 镜像初始化逻辑与 Systemd 不同
- **解决**：在 skip-grant-tables 启动后，使用 INSERT 直接操作用户表，指定 `mysql_native_password`

## 经验教训

1. MySQL 8 的认证机制有较大变化，迁移时需注意版本差异
2. Docker 化迁移需要额外关注启动参数和初始化脚本
3. 数据备份后务必验证恢复流程
