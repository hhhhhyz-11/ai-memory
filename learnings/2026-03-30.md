# 2026-03-30 学习总结

## 技术知识

### 1. GitLab 升级最佳实践
- 物理备份 `data/conf/logs` 到独立路径
- 生产环境升级建议分步骤：11.1.4 → 11.11.8 → 12.0.12 → 12.1.17 → ... → 16.1.8
- 每个大版本升级后需验证 `gitlab:check` 全部通过

### 2. MySQL 从库复制优化
- 多线程复制参数：`slave_parallel_workers=4`, `slave_parallel_type=LOGICAL_CLOCK`
- 从库延迟常见原因：磁盘 I/O 打满、其他服务争抢（MinIO）
- 建议调大 `innodb_buffer_pool_size`（从 12G 建议调到 100G）

### 3. WSL 网络与 Windows 端口映射
- WSL IP 网段与宿主机不同，需用 `netsh portproxy` 做端口映射
- 适用于让宿主机服务访问 WSL 内服务

### 4. Prometheus 告警转发方案
- Alertmanager 无直接飞书 Webhook 能力
- 需用 Python 中转服务处理

## 经验教训

1. **磁盘 I/O 争抢问题**：从库与 MinIO 共用磁盘会导致延迟，部署时需分离
2. **GitLab 恢复需两次确认**：`gitlab-restore` 时需输入两次 `yes`
3. **升级版本跨度**：生产环境不建议跨越大版本，容易出问题
