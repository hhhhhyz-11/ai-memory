## 技术知识

### 1. OpenClaw Node 节点管理
- **问题**：H 节点掉线
- **原因**：Windows 环境下环境变量 `OPENCLAW_GATEWAY_TOKEN` 未设置
- **解决**：设置 token 后节点恢复正常

### 2. SonarQube + GitLab CI 增量扫描
- **需求**：MR 合并到 uat 时触发增量代码扫描
- **配置**：推送代码到 feature 分支时自动触发 SonarQube 扫描
- **关键参数**：`-Dsonar.newCode.referenceBranch=uat` 进行增量对比
- **限制**：GitLab 11.1.4 不支持 `rules` 和 `only: merge_requests`，需用 shell 脚本判断

### 3. Jenkins workspace 命名问题

- **原因**：workspace 实际名称是 `uav-defence-web@2`（之前构建被强制中止导致）
- **解决**：修改 yml 配置使用正确的 workspace 路径

### 4. Jira URL 拼写错误排查
- **问题**：访问 `brwse/BIKE-11560` 链接失败

## 踩坑记录

### 1. Jenkins workspace 名称异常
- **问题**：rsync 找不到 dist 目录
- **原因**：之前构建被强制中止，导致 workspace 名称变成 `uav-defence-web@2`
- **解决**：使用正确的 workspace 路径

### 2. Jira URL 拼写错误
- **问题**：`brwse` 应该是 `browse`
- **影响**：用户无法访问 Jira 链接

### 3. GitLab CI 老版本限制
- **问题**：GitLab 11.1.4 不支持现代 CI 语法
- **解决**：用 shell 脚本判断分支和 MR 状态

## 经验教训

1. **节点管理**：Windows 环境下需确保环境变量 `OPENCLAW_GATEWAY_TOKEN` 已设置
2. **Jenkins workspace**：构建被强制中止可能导致 workspace 名称变化，需注意 `@` 后缀
3. **GitLab CI 兼容**：老版本 GitLab 使用传统 CI 语法，配置时需考虑版本兼容性
4. **增量扫描**：使用 `sonar.newCode.referenceBranch` 参数指定对比分支
