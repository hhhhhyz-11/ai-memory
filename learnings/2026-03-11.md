## 技术知识

### 1. 定时任务动态提示词
- 定时任务 payload 里的日期不能写死，要用动态描述
- 0点执行的任务应该读取"昨天"的日志（因为任务是0点执行，应总结前一天）

### 2. GitLab CI + SonarQube 流水线优化
- **sonar-scanner-cli 扫描 Java** 需要 `sonar.java.binaries` 指定编译产物路径
- **单一 job 优于多 stage**：CI 分成 build + sonar 两个 stage 时，sonar 阶段没有 .class 文件导致报错
- **Maven 优化参数**：
  - `-T 2C` 并行编译
  - `-Dmaven.test.skip=true` 跳过测试
  - `-Dsonar.exclusions` 排除目录

### 3. SonarQube Issues 批量导出
- **问题**：SonarQube 页面导出 PDF 超过 10000 条限制
- **方案**：导出 Protobuf 格式，用 Java 工具转 CSV
- **完整流程**：
  1. SonarQube 页面手动导出项目（Protobuf 格式）→ 生成 .zip
  2. Jenkins Pipeline 调用 shell 脚本
  3. 脚本检测 zip → 解压 → 删除源文件 → 转 CSV → 输出到 csv 目录
  4. Nginx 搭建文件服务器，提供下载入口

## 踩坑记录

### 1. 定时任务日期写死
- **问题**：cron 任务提示词写死日期"今天是2026-03-09"，导致每次都读3月9日
- **解决**：改为动态描述"读取昨天的日志"

### 2. GitLab CI 多 stage 导致 Sonar 分析失败
- **问题**：CI 分成 build + sonar 两个 stage，sonar 阶段没有 .class 文件导致报错
- **原因**：多 stage 情况下编译产物不会自动传递到下一个 stage
- **解决**：换回单一 job，Maven 编译和 Sonar 分析一起执行

## 经验教训

1. **定时任务设计**：任何时间相关的任务都用动态描述，避免写死日期
2. **SonarQube Java 扫描**：必须指定 `sonar.java.binaries` 指向编译产物目录
3. **CI 流水线设计**：如果后序步骤需要编译产物，优先使用单一 job 或 artifact 传递
4. **大数据量导出**：当页面导出有限制时，考虑 API 或其他格式（Protobuf）导出
