# 2026-03-18 学习总结

## 技术知识

### Nginx 代理转发
- `proxy_pass` 尾部斜杠的行为差异：
  - 有斜杠（如 `proxy_pass https://example.com/path/`）：Nginx 会替换 location 匹配的路径部分
  - 无斜杠（如 `proxy_pass https://example.com`）：保留原始请求路径
- 多级转发场景：测试机 → 生产机 → 第二层转发 → 后端服务器

### GitLab CI 自动合并
- 使用 PAT（Personal Access Token）调用 GitLab API
- Merge API 端点：`PUT /projects/:id/merge_requests/:iid/merge`
- 合并前置条件：MR 状态为 `opened` 且 `can_be_merged` 为 true
- 常用工具：Alpine 镜像 + curl + jq

## 踩坑记录

### Nginx 转发 404 问题
- **问题**：测试域名转发到生产域名时返回 404
- **原因**：`proxy_pass` 路径配置错误，尾部斜杠导致路径重复
- **解决**：去掉 `proxy_pass` 中的尾部路径，保留基础域名

## 经验教训

1. Nginx 代理配置要注意尾部斜杠，处理多级转发时要格外小心路径拼接
2. GitLab CI 中使用 PAT 时要注意 token 权限，需要有 Merge 权限才能自动合并
3. 调用外部 API 前先检查状态码和返回值，确保合并条件满足再执行
