# 2026-03-09 技术博客：从零搭建 OpenClaw AI 助手节点

## 前言

最近我完成了一件很有趣的事情：把一台 Windows 电脑配置成 AI 助手 OpenClaw 的执行节点。这意味着我可以远程让 AI 操作我远端的电脑执行命令。

今天就来说说这个过程，和其中涉及的技术原理。

---

## 什么是 OpenClaw 节点？

OpenClaw 是一个 AI 助手框架，类似于 Claude Desktop，但更强大、更灵活。它支持：

- **多平台**: macOS、Windows、Linux、iOS、Android
- **多通道**: Telegram、Discord、WhatsApp、Webchat 等
- **远程执行**: 通过"节点"机制，AI 可以远程执行命令

节点（Node）是连接到 OpenClaw Gateway 的远程设备。配对后，AI 就能在这个设备上：
- 运行 shell 命令
- 截屏/录屏
- 发送通知
- 浏览器自动化

## 搭建过程

### 1. 配置 Gateway 外部访问

默认情况下，OpenClaw Gateway 只监听 `127.0.0.1`（本机）。要让它接受外部连接，需要修改配置：

```json
{
  "gateway": {
    "bind": "custom",
    "customBindHost": "0.0.0.0",
    "port": 18789
  }
}
```

**关键点**：
- `bind: "custom"` + `customBindHost: "0.0.0.0"` 允许任意网络接口访问
- 端口 18789 是默认端口
- 需要 Token 认证保证安全

### 2. 节点配对流程

```
[节点] --WebSocket--> [Gateway]
        |
        v
   发送配对请求
        |
        v
   [用户审批] --> openclaw devices approve <requestId>
        |
        v
   配对成功，节点状态变为 "connected"
```

**重要命令**：
```bash
# 查看配对请求
openclaw devices list

# 批准配对
openclaw devices approve <requestId>

# 查看节点状态
openclaw nodes status
```

### 3. Windows 节点特殊配置

Windows 节点默认不允许远程执行命令，需要配置 exec-approvals：

```json
{
  "version": 1,
  "defaults": {
    "mode": "allow"
  }
}
```

这个文件位于：`C:\Users\<用户名>\.openclaw\exec-approvals.json`

**为什么需要？**
这是安全机制。AI 虽然好用，但毕竟是个"可能犯错的助手"。通过 exec-approvals，可以：
- 控制哪些命令可以执行
- 设置是否需要每次确认
- 完全禁用远程执行

### 4. 远程执行命令

配对成功后，就可以远程执行了：

```bash
# 在 Windows 上打开浏览器访问百度
openclaw nodes run --node H --raw 'start https://www.baidu.com'

# 查看节点信息
openclaw nodes describe --node H
```

## 技术架构分析

### WebSocket 连接

节点通过 WebSocket 连接到 Gateway，保持长连接。这比 HTTP 轮询更高效：

- **实时性**: 服务器可以随时推送消息
- **双向通信**: 节点可以主动发送事件
- **低开销**: 不用每次请求都建立 TCP 连接

### 安全模型

1. **Token 认证**: 每个连接都需要有效的 Token
2. **配对机制**: 新设备需要用户审批
3. **执行审批**: 本地可以控制远程命令的执行策略
4. **命令白名单**: Gateway 端可以限制允许的命令

### 可扩展性

- 一个 Gateway 可以连接多个节点
- 节点可以是任意平台（Windows、macOS、Linux、移动设备）
- 每个节点可以暴露不同的能力（浏览器、shell、摄像头等）

## 应用场景

1. **远程办公**: 在手机上给家里电脑发命令执行任务
2. **自动化**: 定时触发远程脚本执行
3. **跨平台**: 在 Linux 服务器上调用 macOS 特有的命令
4. **物联网**: 手机的 AI 助手控制家庭服务器

## 总结

通过这次搭建，我深入理解了：

1. **Gateway-Agent 架构**: 分离了"大脑"（Gateway）和"四肢"（节点）
2. **WebSocket 长连接**: 适合需要实时交互的场景
3. **安全分层**: 认证 -> 配对 -> 审批 -> 执行，一层层把关
4. **远程执行的艺术**: 既要灵活，又要安全

AI 助手不应该只停留在"聊天"，它应该能真正帮你做事。节点机制让这成为了可能。
