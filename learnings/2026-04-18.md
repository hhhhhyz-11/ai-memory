### 1. OpenClaw 心跳机制（Heartbeat）
- **用途**：约30分钟自动发送系统消息，检查待办事项
- **配置**：默认 `every: "30m"`
- **调整方式**：`openclaw config set agents.defaults.heartbeat.every "2h"`
- **响应方式**：读取 HEARTBEAT.md，按规则检查，无事回复 `HEARTBEAT_OK`

## 踩坑记录

### 1. Kingbase kpatch 升级路径问题 ⚠️
- **问题**：kpatch 路径不能包含 KESRealPro
- **错误**：`./kpatch -k /home/kingbase/KESRealPro/V009R002C013/Server ...`
- **报错**：`kingbase_path contains:KESRealPro, should not use with:-k *KESRealPro*`
- **解决**：改用软链接路径 `-k /home/kingbase/Server`

### 2. Kingbase 必须先启动才能执行 kpatch check
- **原因**：kpatch 会检查 kingbase 状态
- **报错**：`kingbase not running`
- **解决**：先 `sys_ctl -D /home/kingbase/data start`

### 3. PID 文件残留导致启动失败
- **报错**：`lock file "kingbase.pid" already exists`
- **解决**：`sys_ctl -D /home/kingbase/data stop` 清理后再启动

## 经验教训

1. **定时任务配置**：注意参数完整性，避免缺参数导致的任务报错
2. **Gateway 安全**：绑定地址为 0.0.0.0 时需注意安全风险
3. **数据库升级**：必须先启动服务再执行检查工具
4. **路径规范化**：避免路径中包含版本号等动态内容，使用软链接统一路径
