# 2026-03-31 学习总结

## 技术知识

### GitLab CI
- `tags: [yyz]` 用于指定 runner 标签，限制 job 在特定 runner 上运行
- `${CI_COMMIT_REF_SLUG}` 是 GitLab CI 预定义变量，将 commit 引用转换为 URL 安全的 slug 格式

### OpenClaw 飞书集成
- 国内版飞书 App 接入流程：App ID 配置 + 用户授权配对
- 飞书用户唯一标识格式：`ou_xxx`

### MCP 协议
- MCP（Model Context Protocol）可对接 Prometheus 实现 AI 智能运维监控
- 适用于自动化告警分析和故障诊断场景

### RocketMQ 集群
- 5.x 版本 ACL 2.0 认证格式与旧版不同
- 多集群 namesrv 地址用分号分隔：`addr1:9876;addr2:9876`
- broker-c 需在所有节点 broker 停止后才能正确激活

### K8S 节点管理
- 下线节点流程：drain → delete node

### PostgreSQL 迁移
- 使用 pg_dumpall 导出全量数据
- Docker 部署需配置数据目录映射

## 经验教训

1. **RocketMQ 集群迁移**：新增 broker 后旧 broker 需完全停止，否则新 broker 会因冲突显示 `ACTIVATED: false`
2. **MCP 协议**：是标准化 AI 与外部系统交互的新兴协议，值得深入研究
3. **GitLab CI runner 标签**：多个 job 共享 tags 时可统一配置在 stage 级别
