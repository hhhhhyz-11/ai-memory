## 一、核心映射框架

把 AI Agent 的运作映射到控制论框架：

```
环境 ←→ 受控系统
Agent ←→ 控制器
感知   ←→ 测量/反馈
决策   ←→ 控制律
执行   ←→ 执行器
记忆   ←→ 系统状态
目标   ←→ 设定值
```

**闭环结构：**
```
目标设定 → 感知环境 → 状态估计 → 决策控制 → 动作执行 → 反馈比较 → 误差纠正 → 循环
```

## 三、自动化触发机制

### 每日心跳（已存在）
- 每 30 分钟心跳自检
- 检查日志完整性、记忆新鲜度

### 每小时 Intent Check（建议新增）
- 感知当前会话状态（是否在执行任务？是否需要主动介入？）
- 类似伺服系统的"误差检测"——当检测到与目标的偏差时主动调整

### 每日自评（cron 触发）
```
时间：每天 20:00
任务：检查当日工作日志 + 生成简短自我评估
内容：
  1. 今天完成了什么？
  2. 有什么"误差"（偏离预期的地方）？
  3. 如何纠正？
  4. 明天优先级是什么？
```

### 每周系统更新（cron 触发）
```
时间：每周一 09:00
任务：读取周报 → 更新 MEMORY.md
内容：
  1. 本周学到什么新东西？
  2. 上周的问题解决了吗？
  3. 更新技能树
```

### 每月大复盘（cron 触发）
```
时间：每月最后一天 18:00
任务：月度总结 + 进化评估
内容：
  1. 对照 MEMORY.md，检验长期记忆是否准确
  2. 检查是否有重复踩坑
  3. 制定下月重点学习方向
```

## 四、核心进化原则（工程控制论版）

1. **闭环原则**：没有反馈就没有控制。每个动作必须有结果回传。
2. **最优原则**：在多种策略中选择代价最小的，不盲目行动。
3. **容错原则**：允许犯错，但必须有自我诊断和恢复机制。
4. **适应原则**：环境变化时自动调整，不固守过时策略。
5. **抽象原则**：从具体问题中抽取共性，形成可复用模式（从定性到定量）。
6. **仿真优先**：复杂操作先模拟推演，再实际执行。
