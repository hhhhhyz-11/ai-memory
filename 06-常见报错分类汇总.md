# KingbaseES V9R2C14 常见报错分类汇总（初级）

> 适用版本：KingbaseES V9R2C14
> 参考：https://docs.kingbase.com.cn/cn/KES-V9R2C14/question/

---

## 一、连接类错误

### 1.1 连接被拒绝

```
错误信息：
could not connect to server: Connection refused
Is the server running on host "xxx" and accepting
TCP/IP connections on port 54321?

原因：
1. 数据库服务未启动
2. 端口号错误
3. 防火墙阻止
4. listen_addresses 配置不当
```

**排查步骤：**

```bash
# 1. 检查服务是否运行
ps -ef | grep kingbase

# 2. 检查端口监听
netstat -tlnp | grep 54321
ss -tlnp | grep 54321

# 3. 检查防火墙
firewall-cmd --list-ports
iptables -L -n

# 4. 检查配置
grep listen_addresses /data/kingbase/data/kingbase.conf
grep port /data/kingbase/data/kingbase.conf
```

**解决方案：**

```bash
# 启动数据库
./sys_monitor start -D /data/kingbase/data

# 修改 kingbase.conf
sed -i "s/listen_addresses = '.*'/listen_addresses = '*'/" /data/kingbase/data/kingbase.conf

# 重新加载配置
./sys_monitor reload -D /data/kingbase/data
```

---

### 1.2 认证失败

```
错误信息：
FATAL: password authentication failed for user "xxx"
FATAL: SCRAM authentication failed for user "xxx"

原因：
1. 用户名或密码错误
2. pg_hba.conf 认证方式不匹配
3. 用户不存在
```

**排查步骤：**

```bash
# 1. 检查 pg_hba.conf 认证方式
cat /data/kingbase/data/pg_hba.conf

# 2. 确认用户存在
./ksql -U system -d test -p 54321 -c "\du"

# 3. 查看日志中的详细错误
tail -50 /data/kingbase/data/log/kingbase-*.log
```

**解决方案：**

```sql
-- 重置用户密码
ALTER USER username WITH PASSWORD 'new_password';

-- 修改 pg_hba.conf 为 trust 方式（测试环境）
-- host all all 0.0.0.0/0 trust
```

---

### 1.3 用户权限不足

```
错误信息：
ERROR: permission denied for table "xxx"
ERROR: must be owner of table "xxx"
ERROR: must be member of role "xxx"

原因：当前用户没有操作对象的权限
```

**解决方案：**

```sql
-- 授予表权限
GRANT SELECT, INSERT, UPDATE, DELETE ON TABLE my_table TO my_user;

-- 授予所有表权限
GRANT ALL PRIVILEGES ON ALL TABLES IN SCHEMA public TO my_user;

-- 授予 schema 权限
GRANT ALL ON SCHEMA public TO my_user;

-- 授予表拥有者
ALTER TABLE my_table OWNER TO new_owner;
```

## 二、SQL 执行错误

### 2.1 语法错误

```
错误信息：
ERROR: syntax error at or near "xxx"
LINE 1: xxx

原因：SQL 语句语法错误
```

**排查：** 仔细检查 SQL 语句，特别是：
- 关键字拼写
- 引号使用（单引号 vs 双引号）
- 括号匹配
- 逗号分隔

**示例：**

```sql
-- 错误
SELECT * FORM my_table WHERE id = 1;

-- 正确
SELECT * FROM my_table WHERE id = 1;
```

---

### 2.2 数据类型错误

```
错误信息：
ERROR: operator does not exist: integer = text
HINT: No operator matches the given name and argument types.
You might need to add explicit type casts.

原因：数据类型不匹配
```

**解决方案：**

```sql
-- 显式类型转换
SELECT * FROM my_table WHERE id = '1'::integer;
SELECT * FROM my_table WHERE name = 'test';  -- text 不需要转换

-- 或者使用 CAST
SELECT * FROM my_table WHERE id = CAST('1' AS integer);
```

---

### 2.3 主键/唯一键冲突

```
错误信息：
ERROR: duplicate key value violates unique constraint "xxx"
DETAIL: Key (id)=(1) already exists.

原因：插入或更新的数据违反了唯一约束
```

**排查：**

```sql
-- 查看冲突的键值
SELECT * FROM my_table WHERE id = 1;

-- 查看约束定义
\d my_table
```

---

### 2.4 外键约束失败

```
错误信息：
ERROR: insert or update on table "child" violates foreign key constraint "xxx"
DETAIL: Key (parent_id)=(999) is not present in the parent table "parent".

原因：子表引用的父表记录不存在
```

**解决方案：**

```sql
-- 先插入父表记录
INSERT INTO parent (id, name) VALUES (999, 'Parent Name');

-- 或者修改子表的外键值
INSERT INTO child (id, parent_id) VALUES (1, 现有父表ID);
```

---

### 2.5 空值约束失败

```
错误信息：
ERROR: null value in column "xxx" violates not-null constraint
DETAIL: Failing row contains (null, ...).

原因：向 NOT NULL 列插入了 NULL 值
```

**解决方案：**

```sql
-- 检查并提供非空值
INSERT INTO my_table (id, name) VALUES (1, 'value');

-- 或者修改表结构允许空值
ALTER TABLE my_table ALTER COLUMN name DROP NOT NULL;
```

## 三、数据库操作错误

### 3.1 表空间不足

```
错误信息：
ERROR: could not extend relation ...: No space left on device
ERROR: could not write to file "xxx": No space left on device

原因：磁盘空间不足
```

**排查：**

```bash
# 检查磁盘空间
df -h

# 检查数据目录所在分区
df -h /data/kingbase/data

# 检查文件大小限制
df -i  # inode 检查
```

**解决方案：**

```bash
# 清理不需要的文件
rm -rf /tmp/xxx
find /var/log -type f -size +100M -exec truncate -s 0 {} \;

# 扩展磁盘或数据目录
# 添加新表空间
```

```sql
-- 查看各表空间使用情况
SELECT spcname,
       pg_size_pretty(pg_tablespace_size(spcname))
FROM sys_tablespace;
```

---

### 3.2 连接数超限

```
错误信息：
ERROR: remaining connection slots are reserved for non-replication superuser connections
FATAL: sorry, too many clients already

原因：超过了 max_connections 限制
```

**排查：**

```sql
-- 查看当前连接数
SELECT count(*) FROM sys_stat_activity;

-- 查看各用户连接数
SELECT usename, count(*) FROM sys_stat_activity GROUP BY usename;
```

**解决方案：**

```sql
-- 杀掉空闲连接
SELECT pg_terminate_backend(pid)
FROM sys_stat_activity
WHERE state = 'idle'
AND query_start < current_timestamp - interval '10 minutes';

-- 临时增加最大连接数
ALTER SYSTEM SET max_connections = 2000;
-- 需要重启生效
```

---

### 3.3 事务回滚

```
错误信息：
ERROR: current transaction is aborted, commands ignored until end of transaction block

原因：事务中出现错误后，后续命令被忽略
```

**解决方案：**

```sql
-- 必须回滚当前事务
ROLLBACK;

-- 然后重新开始新事务
BEGIN;
-- 重试之前的操作
COMMIT;
```

## 四、系统类错误

### 4.1 数据库无法启动

```
错误信息：
FATAL: could not open lock file "/data/kingbase/data/postmaster.pid": No such file or directory
FATAL: lock file "xxx" is already being used

原因：
1. 数据目录不存在或权限错误
2. 已有实例在运行
3. 上次异常关闭遗留锁文件
```

**排查：**

```bash
# 检查进程
ps -ef | grep kingbase

# 检查 PID 文件
ls -la /data/kingbase/data/postmaster.pid 2>/dev/null

# 检查数据目录权限
ls -la /data/kingbase/data/
```

**解决方案：**

```bash
# 如果进程存在但PID文件丢失，重新创建
touch /data/kingbase/data/postmaster.pid

# 如果端口被占用，杀掉进程
kill $(lsof -t -i:54321)

# 如果是锁文件问题，删除锁文件后重启
rm /data/kingbase/data/postmaster.pid
rm /data/kingbase/data/postmaster.opts
./sys_monitor start -D /data/kingbase/data
```

---

### 4.2 WAL 日志错误

```
错误信息：
FATAL: could not open file "xxx": No such file or directory
ERROR: WAL archive contains invalid header

原因：WAL 文件丢失或损坏
```

**解决方案：**

```bash
# 检查归档配置
grep archive /data/kingbase/data/kingbase.conf

# 如果归档出现问题，跳过损坏的 WAL
# 编辑 recovery.conf 或修改配置
```

---

### 4.3 数据库损坏

```
错误信息：
ERROR: could not read block xxx of relation base/xxx/xxx: read failed

原因：数据文件损坏
```

**排查：**

```sql
-- 检查表是否损坏
SELECT * FROM sys_class WHERE relname = 'my_table';
SELECT sys_check_pgdir('/data/kingbase/data');

-- 查看损坏的具体页面
SELECT * FROM sys_filedump('base/16384/12345');
```

**解决方案：**

```bash
# 1. 尝试修复（如果可能）
./ksql -U system -d test -p 54321 -c "REINDEX TABLE损坏表;"

# 2. 从备份恢复
# 参考备份恢复文档

# 3. 使用 sys_rman 进行修复
./sys_rman restore ...
```

## 五、错误代码速查表

| 错误代码 | 说明 | 常见原因 |
|---------|------|---------|
| `28P01` | 认证失败 | 密码错误 |
| `28000` | 无效的授权 | pg_hba.conf 配置 |
| `42P01` | 表不存在 | schema 未指定或表名错误 |
| `23505` | 唯一约束冲突 | 主键/唯一键重复 |
| `23503` | 外键约束失败 | 父表记录不存在 |
| `23502` | 非空约束失败 | NULL 值插入 |
| `53200` | 内存不足 | work_mem 太小 |
| `53000` | 磁盘空间不足 | 数据目录满 |
| `08000` | 连接异常 | 网络问题 |
| `08006` | 连接断开 | 数据库崩溃 |

## 七、快速诊断命令

```bash
# 1. 检查服务状态
./sys_monitor status -D /data/kingbase/data

# 2. 检查连接
./ksql -U system -d test -p 54321 -c "SELECT 1;"

# 3. 检查版本
./ksql -U system -d test -p 54321 -c "SELECT version();"

# 4. 检查表空间
./ksql -U system -d test -p 54321 -c "\db"

# 5. 检查当前连接
./ksql -U system -d test -p 54321 -c "SELECT count(*) FROM sys_stat_activity;"

# 6. 检查数据库大小
./ksql -U system -d test -p 54321 -c "SELECT pg_database_size(current_database());"

# 7. 检查配置
./ksql -U system -d test -p 54321 -c "SHOW all;" | head -50
```
