# KingbaseES V9R2C14 单机部署全流程

> 适用版本：KingbaseES V9R2C14
> 参考：https://docs.kingbase.com.cn/cn/KES-V9R2C14/deployment/

---

## 一、环境准备

### 1.1 系统要求

| 项目 | 要求 |
|------|------|
| CPU | x86_64 架构，推荐 4 核以上 |
| 内存 | 推荐 8GB 以上，生产环境 16GB+ |
| 磁盘 | 推荐 SSD，至少 50GB 可用空间 |
| 操作系统 | Linux（Red Hat / CentOS / 麒麟 / UOS 等） |
| 文件系统 | ext4 / xfs |
| 网络 | 千兆网络 |

### 1.2 检查依赖包

```bash
# CentOS/RHEL 系统检查并安装依赖
yum install -y \
  libicu \
  libnsl \
  ncurses-libs \
  openssl \
  pam \
  readline \
  zlib

# 检查版本
cat /etc/redhat-release
uname -r
```

### 1.3 创建用户和目录

```bash
# 创建 kingbase 用户（建议使用普通用户运行）
useradd -m -d /home/kingbase -s /bin/bash kingbase
passwd kingbase

# 创建安装目录
mkdir -p /opt/Kingbase/ES/V9R2C14
mkdir -p /data/kingbase/data
mkdir -p /data/kingbase/wal
chown -R kingbase:kingbase /data/kingbase
chown -R kingbase:kingbase /opt/Kingbase
```

### 1.4 系统参数调整

```bash
# /etc/sysctl.conf
cat >> /etc/sysctl.conf << 'EOF'
```

## 二、安装 KingbaseES

### 2.1 上传安装包

```bash
# 将安装包上传到服务器，假设放在 /tmp 目录
ls -lh /tmp/KingbaseES_V9R2C14_Linux.tar.gz
```

### 2.2 解压安装包

```bash
cd /tmp
tar -xzf KingbaseES_V9R2C14_Linux.tar.gz
cd KingbaseES_V9R2C14_Linux
ls -la
# 应该看到 install.py 或 setup.bin
```

### 2.3 交互式安装（图形界面）

```bash
# 如果有图形界面
python3 install.py

# 或使用 xhost + && ./setup.bin
```

### 2.4 静默安装（推荐生产环境）

```bash
# 创建 response.rsp 配置文件
cat > /tmp/kingbase.rsp << 'EOF'
# KingbaseES Installation Response File
KB_INSTALL=TRUE
KB_INSTALL_PATH=/opt/Kingbase/ES/V9R2C14
KB_INSTALL_MODE=TEMPLATE
EOF

# 执行静默安装
python3 install.py -f /tmp/kingbase.rsp -s
```

### 2.5 安装验证

```bash
# 检查安装目录
ls -la /opt/Kingbase/ES/V9R2C14/

# 检查版本
/opt/Kingbase/ES/V9R2C14/Server/bin/kingbase --version
```

## 三、初始化数据库实例

### 3.1 切换到 kingbase 用户

```bash
su - kingbase
```

### 3.2 初始化数据目录

```bash
cd /opt/Kingbase/ES/V9R2C14/Server/bin

# 初始化（单机模式）
./sys_monitor initdb \
  -D /data/kingbase/data \
  -U system \
  -W your_password \
  --encoding=UTF8 \
  --locale=C

# 参数说明：
# -D : 数据目录（必须提前创建并授权）
# -U : 超级用户名称（默认为 system）
# -W : 超级用户密码
# --encoding : 编码（推荐 UTF8）
# --locale    : 区域设置
```

### 3.3 修改配置参数

```bash
# 编辑 postgresql.conf
cat > /data/kingbase/data/kingbase.conf << 'EOF'
# === 基本参数 ===
listen_addresses = '*'
port = 54321
max_connections = 1000
superuser_reserved_connections = 10

# === 内存参数 ===
shared_buffers = 4GB          # 建议为服务器内存的 1/4
effective_cache_size = 12GB    # 建议为服务器内存的 3/4
work_mem = 256MB
maintenance_work_mem = 512MB
random_page_cost = 1.1
effective_io_concurrency = 200

# === 存储参数 ===
wal_level = replica
max_wal_size = 4GB
min_wal_size = 1GB
wal_buffers = 64MB
checkpoint_completion_target = 0.9

# === 日志参数 ===
logging_collector = on
log_directory = 'log'
log_filename = 'kingbase-%Y-%m-%d.log'
log_rotation_age = 1d
log_rotation_size = 100MB
log_line_prefix = '%t [%p]: [%l-1] user=%u,db=%d,app=%a,client=%h '
log_statement = 'ddl'
log_duration = off
log_min_duration_statement = 1000

# === 归档参数 ===
archive_mode = on
archive_command = 'test ! -f /data/kingbase/wal/%f && cp %p /data/kingbase/wal/%f'

# === 字符集 ===
client_encoding = UTF8
server_encoding = UTF8

# === 其他 ===
shared_preload_libraries = 'sys_kwr'
track_activity_query_size = 2048
update_process_title = off
datestyle = 'iso, ymd'
timezone = 'PRC'
EOF
```

### 3.4 配置 pg_hba.conf（访问控制）

```bash
cat > /data/kingbase/data/pg_hba.conf << 'EOF'
# TYPE  DATABASE        USER            ADDRESS                 METHOD

# 本地超级用户
local   all             system                                  trust

# 本地连接
local   all             all                                     scram-sha-256

# IPv4 远程连接（根据实际需求调整）
host    all             all             0.0.0.0/0               scram-sha-256

# IPv6 远程连接
host    all             all             ::0/0                   scram-sha-256

# 本地 replication
local   replication     all                                     trust
host    replication     all             127.0.0.1/32            trust
host    replication     all             ::1/128                 trust
EOF
```

### 3.5 启动数据库

```bash
# 使用 sys_monitor 启动
./sys_monitor start -D /data/kingbase/data

# 或直接使用 kingbase 命令
./kingbase -D /data/kingbase/data -l /data/kingbase/data/logfile start

# 检查启动状态
./sys_monitor status -D /data/kingbase/data
```

### 3.6 验证数据库

```bash
# 连接测试
./ksql -U system -d test -p 54321

# 或本地 socket 连接
./ksql -U system -d test

# 检查版本
SELECT version();

# 检查数据库列表
\l

# 检查当前连接数
SELECT count(*) FROM sys_stat_activity;
```

## 五、服务管理命令汇总

```bash
# 启动
./sys_monitor start -D /data/kingbase/data

# 停止（推荐使用 smart 模式，强制关闭可能丢数据）
./sys_monitor stop -D /data/kingbase/data -m smart    # 等待所有连接结束
./sys_monitor stop -D /data/kingbase/data -m fast     # 发送 SIGTERM，快速关闭
./sys_monitor stop -D /data/kingbase/data -m immediate # 强制关闭，可能需要 recovery

# 重启
./sys_monitor restart -D /data/kingbase/data

# 重新加载配置（无需重启）
./sys_monitor reload -D /data/kingbase/data

# 查看状态
./sys_monitor status -D /data/kingbase/data

# 查看进程
ps -ef | grep kingbase
```

## 六、配置开机自启动

### 6.1 创建 systemd 服务文件

```bash
# root 用户执行
cat > /etc/systemd/system/kingbase.service << 'EOF'
[Unit]
Description=KingbaseES V9R2C14 Database Server
After=network.target firewalld.service

[Service]
Type=forking
User=kingbase
Group=kingbase
ExecStart=/opt/Kingbase/ES/V9R2C14/Server/bin/sys_monitor start -D /data/kingbase/data
ExecStop=/opt/Kingbase/ES/V9R2C14/Server/bin/sys_monitor stop -D /data/kingbase/data -m fast
ExecReload=/opt/Kingbase/ES/V9R2C14/Server/bin/sys_monitor reload -D /data/kingbase/data
PrivateTmp=true
Restart=on-failure
RestartSec=10

[Install]
WantedBy=multi-user.target
EOF

# 重载 systemd
systemctl daemon-reload

# 启用开机自启
systemctl enable kingbase

# 启动服务
systemctl start kingbase
systemctl status kingbase
```

### 6.2 配置 /etc/rc.local

```bash
# 如果不使用 systemd
cat >> /etc/rc.local << 'EOF'
su - kingbase -c "/opt/Kingbase/ES/V9R2C14/Server/bin/sys_monitor start -D /data/kingbase/data"
EOF
chmod +x /etc/rc.d/rc.local
```

## 八、常见问题排查

```bash

# 检查磁盘空间
df -h

# 检查用户权限
id kingbase
```

```bash

cat /data/kingbase/data/log/*.log
tail -100 /data/kingbase/data/log/kingbase-*.log

# 检查端口是否被占用
netstat -tlnp | grep 54321
ss -tlnp | grep 54321

# 检查数据目录权限

```

### 8.3 连接被拒绝

```bash
# 检查 pg_hba.conf 配置
cat /data/kingbase/data/pg_hba.conf

# 检查 listen_addresses 配置
grep listen_addresses /data/kingbase/data/kingbase.conf

# 重载配置
./sys_monitor reload -D /data/kingbase/data
```
