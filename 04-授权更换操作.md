# KingbaseES V9R2C14 授权更换操作（MAC地址 / 授权码）

> 适用版本：KingbaseES V9R2C14
> 参考：https://docs.kingbase.com.cn/cn/KES-V9R2C14/reference/license/

---

## 一、授权概述

KingbaseES 支持两种授权方式：
1. **MAC 地址授权** — 通过网卡物理地址绑定授权
2. **授权码授权** — 通过金仓提供的授权码文件激活

---

## 二、MAC 地址授权

### 2.1 查看 MAC 地址

```bash
# 查看所有网卡的 MAC 地址
ip link show
ifconfig -a
cat /sys/class/net/*/address

# 确认需要绑定的网卡名称（如 eth0, ens33, enp0s3 等）
ip addr show
```

### 2.2 获取机器码

```bash
# 切换到 kingbase 用户
su - kingbase
cd /opt/Kingbase/ES/V9R2C14/Server/bin

# 查看许可证相关信息
ls -la *.lic 2>/dev/null || echo "No license files found"

# 查看当前授权状态（连接数据库）
./ksql -U system -d test -p 54321 -c "SELECT * FROM sys_license_info();"
```

### 2.3 申请 MAC 授权

1. 记录服务器 MAC 地址
2. 联系金仓技术支持，提供 MAC 地址申请授权文件
3. 获得 `.lic` 授权文件

### 2.4 安装授权文件

```bash
# 方法一：放置在安装目录
cp your_license.lic /opt/Kingbase/ES/V9R2C14/Server/lib/
chown kingbase:kingbase /opt/Kingbase/ES/V9R2C14/Server/lib/your_license.lic

# 方法二：放置在数据目录
cp your_license.lic /data/kingbase/data/
chown kingbase:kingbase /data/kingbase/data/your_license.lic

# 方法三：KingbaseLAC 管理平台授权（推荐）
# 参考 KingbaseLAC 部署文档
```

### 2.5 验证授权

```sql
-- 连接数据库查看授权状态
./ksql -U system -d test -p 54321

-- 查看授权信息
SELECT * FROM sys_license_info();
SELECT * FROM sys_license_info_details();

-- 检查授权是否有效
SELECT sys_verify_license();
```

### 2.6 授权相关视图

```sql
-- 查看详细的授权信息
\d sys_license_info
\d sys_license_info_details

-- 典型授权信息输出
SELECT
    lic_type,        -- 授权类型
    lic_version,     -- 授权版本
    lic_expiredate,  -- 过期日期
    lic_maxusers,    -- 最大用户数
    lic_curusers,    -- 当前用户数
    lic_status       -- 授权状态
FROM sys_license_info;
```

## 三、授权码授权

### 3.1 授权码获取

授权码通常由金仓提供，是一个字符串形式的授权文件，例如：
```
KINGBASE:LICENSE:V9:XXXX-XXXX-XXXX-XXXX:MA=xx:xx:xx:xx:xx:xx:EX=20301231:USERS=100
```

### 3.2 安装授权码

#### Linux 环境

```bash
su - kingbase

# 方法一：写入配置文件
cat > /data/kingbase/data/sys_license.lic << 'EOF'
KINGBASE:LICENSE:V9:XXXX-XXXX-XXXX-XXXX:MA=xx:xx:xx:xx:xx:xx:EX=20301231:USERS=100
EOF
chown kingbase:kingbase /data/kingbase/data/sys_license.lic

# 方法二：写入 /opt/Kingbase/ES/V9R2C14/Server/lib/
cat > /opt/Kingbase/ES/V9R2C14/Server/lib/sys_license.lic << 'EOF'
KINGBASE:LICENSE:V9:XXXX-XXXX-XXXX-XXXX:MA=xx:xx:xx:xx:xx:xx:EX=20301231:USERS=100
EOF
chown kingbase:kingbase /opt/Kingbase/ES/V9R2C14/Server/lib/sys_license.lic
```

#### Windows 环境

```powershell
# 将授权码文件保存为：
# C:\Program Files\KingbaseES\ES\V9R2C14\Server\lib\sys_license.lic
# 或
# C:\ProgramData\Kingbase\ES\V9R2C14\sys_license.lic
```

### 3.3 KingbaseLAC 授权管理

KingbaseLAC（License And Configuration）是金仓提供的图形化授权管理工具：

```bash
# 启动 KingbaseLAC 管理界面
# 通常通过浏览器访问：https://hostname:8889

# 或命令行工具
cd /opt/Kingbase/ES/V9R2C14/Server/bin

# 查看授权状态
./kingbase -C license show

# 导入授权码
./kingbase -C license import /path/to/your_license.lic
```

## 四、授权更换操作

### 4.1 MAC 地址变更（换网卡）

```sql
-- 如果 MAC 地址变更，需要重新申请授权

-- 1. 查看当前授权信息，记录旧 MAC
SELECT * FROM sys_license_info();

-- 2. 停止数据库
cd /opt/Kingbase/ES/V9R2C14/Server/bin
./sys_monitor stop -D /data/kingbase/data

-- 3. 更换网卡后，获取新 MAC 地址
ip link show

-- 4. 联系金仓申请新的授权文件

-- 5. 安装新授权文件
cp new_license.lic /opt/Kingbase/ES/V9R2C14/Server/lib/
chown kingbase:kingbase /opt/Kingbase/ES/V9R2C14/Server/lib/new_license.lic

-- 6. 重启数据库
./sys_monitor start -D /data/kingbase/data

-- 7. 验证新授权
./ksql -U system -d test -p 54321 -c "SELECT * FROM sys_license_info();"
```

### 4.2 授权码更换

```bash
# 1. 备份当前授权文件
cp /data/kingbase/data/sys_license.lic /data/kingbase/data/sys_license.lic.bak

# 2. 停止数据库
./sys_monitor stop -D /data/kingbase/data

# 3. 替换授权文件
cat > /data/kingbase/data/sys_license.lic << 'EOF'
NEW_LICENSE_CODE_HERE
EOF

# 4. 重启数据库
./sys_monitor start -D /data/kingbase/data

# 5. 验证
./ksql -U system -d test -p 54321 -c "SELECT * FROM sys_license_info();"
```

### 4.3 迁移服务器后授权处理

```sql
-- 服务器 IP 或 MAC 变更后授权失效的处理流程

-- 1. 停止原服务器数据库
./sys_monitor stop -D /data/kingbase/data

-- 2. 获取新服务器 MAC 地址
ip link show

-- 3. 联系金仓申请新授权（提供新 MAC 地址）

-- 4. 在新服务器安装 KingbaseES（确保版本一致）

-- 5. 配置授权文件

-- 6. 启动数据库验证授权

-- 7. 恢复数据（如果是从备份恢复）
# 参考备份恢复相关文档
```

## 五、授权相关错误处理

### 5.1 常见错误及解决方案

| 错误信息 | 原因 | 解决方案 |
|---------|------|---------|
| `license expired` | 授权已过期 | 联系金仓续费，获取新授权码 |
| `license invalid MAC` | MAC 地址不匹配 | 重新申请匹配的授权文件 |
| `license file not found` | 授权文件不存在 | 检查授权文件路径和权限 |
| `license count exceeded` | 超过最大用户数 | 增加用户数授权或限制连接 |
| `invalid license format` | 授权码格式错误 | 确认授权码完整性，重新申请 |
| `license check failed` | 授权验证失败 | 检查系统时间是否正确 |

### 5.2 排查命令

```bash
# 查看数据库启动日志中的授权相关日志
grep -i license /data/kingbase/data/log/*.log

# 查看详细授权错误
cat /data/kingbase/data/log/kingbase-*.log | grep -iE 'license|授权'

# 手动验证授权文件格式
cat /data/kingbase/data/sys_license.lic
```

### 5.3 检查系统时间

```bash
# 授权码通常有过期日期，系统时间错误会导致授权验证失败
# 检查系统时间
date

# 同步时间（如果需要）
ntpdate ntp.api.bz
# 或
chronyc makestep

# 查看授权有效期
cat /data/kingbase/data/sys_license.lic | grep EX=
```

## 六、KingbaseLAC 管理平台

### 6.1 LAC 简介

KingbaseLAC 是金仓授权与配置管理平台，提供：
- 可视化授权管理
- 多节点集中授权
- 授权监控与告警

### 6.2 LAC 服务端部署

```bash
# 详细步骤参考官方文档：
# https://docs.kingbase.com.cn/cn/KES-V9R2C14/reference/license/KingbaseLAC参考/

# 一般流程：
# 1. 安装 LAC 服务端
# 2. 配置数据库节点
# 3. 导入授权文件
# 4. 下发授权到各节点
```

### 6.3 常见 LAC 操作

```bash
# LAC 命令行工具（通常在 KingbaseES bin 目录）
./kingbase -C lac help

# 查看连接状态
./kingbase -C lac status

# 重新加载授权
./kingbase -C lac reload

# 查看授权详细信息
./kingbase -C lac info
```

## 七、注意事项

1. **授权文件安全**：授权文件包含敏感信息，妥善保管，避免泄露
2. **定期检查**：定期检查授权有效期，提前续费
3. **变更记录**：MAC 地址或服务器变更时，记录变更时间和原因
4. **备份授权文件**：更换授权前务必备份原有授权文件
5. **版本匹配**：授权文件与 KingbaseES 版本需匹配
