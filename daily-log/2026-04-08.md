### 📍 问题记录（16:57 更新）

**问题1：目录结构错误**

**错误信息：**
```
[ERROR] [RWC-07455] can not connect to 192.168.56.11 from current node (with super_user 'root', execute_user 'kingbase') by '/home/kingbase/cluster/kingbase/Server/bin/sys_securecmd' on port '8890'
```

**原因：**
- install_dir="/home/kingbase/cluster/kingbase"
- 集群脚本期望目录结构：`${install_dir}/Server/bin/sys_securecmd`
- 实际 bin/include/lib/share 在根目录，未在 Server/ 内

**修复步骤：**

1. 停止 securecmdd（root）：
```bash
cd /home/kingbase/cluster/kingbase/bin/
./sys_HAscmdd.sh stop
```

2. 创建 Server 目录并移动文件：
```bash
cd /home/kingbase/cluster/kingbase/
mkdir -p Server
mv bin include lib share Server/
```

3. 验证：
```bash
ls Server/
# 应显示：bin  include  lib  share
```

**⚠️ 需要在两台节点都执行此修复！**

### 📍 问题记录（17:00 更新）

**问题2：ping 不通网关（trusted_servers）**

**错误信息：**
```
[ERROR] [RWC-08200] Failed to ping trusted_servers on host 192.168.56.1, will do nothing and exit.
```

**原因：**
- 防火墙未放行 ICMP 或阻止了网关通信
- 测试环境常见问题

**解决方案：**

**方案1：关闭防火墙（测试环境）**
```bash
# CentOS 7
systemctl stop firewalld
systemctl disable firewalld

# 或放行 ICMP
firewall-cmd --add-protocol=icmp --permanent
firewall-cmd --reload
```

**方案2：修改 install.conf 中的 trusted_servers 为空或本地 IP**
```
trusted_servers=""
```
或者配置一个实际可达的网关 IP

**⚠️ 两台节点都需要处理！**
