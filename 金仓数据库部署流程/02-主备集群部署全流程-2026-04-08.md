## 四、集群部署

### 1. 创建目录结构（重要！）

**⚠️ 必须先创建 Server 目录，再解压 db.zip**

```bash
mkdir -p /home/kingbase/cluster/kingbase/Server
mkdir -p /home/kingbase/cluster/oracle
```

### 2. 复制并解压 db.zip

**在 KingBase1 上执行：**

```bash
cd /home/kingbase/cluster/kingbase/

# 复制 license.dat 和 db.zip
cp ./kingbase/KESRealPro/V009R002C014/Server/bin/license.dat Server/
cp db.zip Server/

# 复制到 KingBase2
scp Server/license.dat kingbase@192.168.56.12:/home/kingbase/cluster/kingbase/Server/
scp Server/db.zip kingbase@192.168.56.12:/home/kingbase/cluster/kingbase/Server/
```

### 3. 解压 db.zip

**两台节点都需要执行：**

```bash
cd /home/kingbase/cluster/kingbase/Server/
unzip db.zip
```

**解压后目录结构：**
```
/home/kingbase/cluster/kingbase/
└── Server/
    ├── bin/
    ├── include/
    ├── lib/
    ├── share/
    ├── log/
    └── db.zip
```

### 4. 配置 install.conf

**文件路径：** /home/kingbase/cluster/oracle/install.conf

```conf
all_ip=(192.168.56.11 192.168.56.12)
install_dir="/home/kingbase/cluster/kingbase"
zip_package="/home/kingbase/cluster/kingbase/Server/db.zip"
license_file=(license.dat)
db_user="system"
db_password="kingbase"
db_port="54321"
db_auth="scram-sha-256"
db_checksums="yes"
archive_mode="always"
encoding="UTF8"
locale="zh_CN.UTF-8"
trusted_servers="192.168.56.1"
virtual_ip="192.168.56.15/24"
net_device=(eth1 eth1)
net_device_ip=(192.168.56.11 192.168.56.12)
deploy_by_sshd=0
```

### 5. 配置 SSH 互信

```bash
cd /home/kingbase/cluster/kingbase/Server/
sh trust_cluster.sh
```

### 6. 配置 securecmdd 服务

**两台节点都需要执行：**

```bash
cd /home/kingbase/cluster/kingbase/Server/bin/
./sys_HAscmdd.sh init
./sys_HAscmdd.sh start
netstat -tlunp |grep 8890
```

### 7. 验证 securecmdd 互信

```bash
./sys_securecmd root@192.168.56.12 'whoami'
./sys_securecmd kingbase@192.168.56.12 'whoami'
```

### 8. 关闭防火墙（测试环境）

```bash
systemctl stop firewalld
systemctl disable firewalld
```

### 9. 执行集群部署

```bash
su - kingbase
cd /home/kingbase/cluster/oracle
sh cluster_install.sh
```

## 六、常见问题

### 问题1：目录结构错误

**原因：** 未提前创建 Server 目录，db.zip 解压到错误位置

**预防方法：** 严格按照本文档步骤1执行，先创建 Server 目录再解压

**解决：**
```bash
cd /home/kingbase/cluster/kingbase/
mkdir -p Server
mv bin include lib share Server/
```

### 问题2：ping 不通网关

**解决：**
```bash
systemctl stop firewalld
systemctl disable firewalld
```

## 八、集群日常运维命令

### 8.1 查看集群状态

```bash
# 查看集群整体状态（推荐）
cd /home/kingbase/cluster/kingbase/Server/bin
./repmgr cluster show

# 输出示例：
#  ID | Name  | Role    | Status  | Upstream | Location | Priority | Timeline | LSN_Lag
# ----+-------+---------+---------+----------+----------+----------+----------+---------
#   1 | node1 | primary | * running |          | default  | 100      | 1        | 
#   2 | node2 | standby | running  | node1    | default  | 100      | 1        | 0 bytes

# 查看集群服务状态（含 repmgrd）
./repmgr service status

# 查看集群详细信息（含连接字符串）
./repmgr cluster show --verbose
```

### 8.2 查看流复制状态

```bash
# 查看流复制连接信息（主库执行）
./ksql -h localhost -U system -d test -p 54321 -W -c "SELECT * FROM sys_stat_replication;"

# 查看复制槽状态（主库执行）
./ksql -h localhost -U system -d test -p 54321 -W -c "SELECT * FROM sys_replication_slots;"

# 查看备库延迟
./ksql -h localhost -U system -d test -p 54321 -W -c "SELECT client_addr, state, sent_lsn, write_lsn, flush_lsn, replay_lsn FROM sys_stat_replication;"
```

### 8.3 集群启动

```bash
# 启动集群（主库执行）
cd /home/kingbase/cluster/kingbase/Server/bin
./sys_ctl -D /home/kingbase/cluster/kingbase/data start

# 如果用 repmgr 启动（两台节点分别执行）
./repmgr -f /home/kingbase/cluster/kingbase/etc/repmgr.conf node start

# 启动 repmgrd 守护进程（两台节点都执行）
./repmgrd -f /home/kingbase/cluster/kingbase/etc/repmgr.conf
```

### 8.4 集群停止

```bash
# 停止集群（优雅停止，推荐）
./sys_ctl -D /home/kingbase/cluster/kingbase/data stop -m fast

# 强制停止（仅紧急情况）
./sys_ctl -D /home/kingbase/cluster/kingbase/data stop -m immediate

# 通过 repmgr 停止（两台节点都执行）
./repmgr -f /home/kingbase/cluster/kingbase/etc/repmgr.conf node stop
```

### 8.5 集群故障切换

```bash
# 手动故障切换（备库执行）
./repmgr standby switchover -f /home/kingbase/cluster/kingbase/etc/repmgr.conf

# 强制提升备库为主库（主库故障时，在备库执行）
./repmgr -f /home/kingbase/cluster/kingbase/etc/repmgr.conf standby promote

# 查看集群事件日志
./repmgr cluster event
```

### 8.6 单节点启停

```bash
# 停止单节点数据库（该节点执行）
./sys_ctl -D /home/kingbase/cluster/kingbase/data stop -m fast

# 启动单节点数据库（该节点执行）
./sys_ctl -D /home/kingbase/cluster/kingbase/data start

# 查看单节点状态
./repmgr -f /home/kingbase/cluster/kingbase/etc/repmgr.conf node status
```
