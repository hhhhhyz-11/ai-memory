## 一、AutoExplain 慢查询分析

### 1.1 简介

AutoExplain 是 KingbaseES 的自动执行计划解释工具，可以自动记录慢查询的执行计划，无需手动 EXPLAIN。

### 1.2 安装配置

```sql
-- 检查扩展
SELECT * FROM sys_available_extensions WHERE name = 'auto_explain';

-- 安装（如果已安装，跳过）
CREATE EXTENSION IF NOT EXISTS auto_explain;
```

### 1.3 配置参数

```sql
-- kingbase.conf 中配置
-- 或在会话中动态设置

-- 启用 AutoExplain
SET auto_explain.enabled = on;

-- 设置记录阈值（毫秒），超过此时间的查询会记录执行计划
SET auto_explain.log_min_duration = '1000';  -- 1 秒

-- 是否记录嵌套语句的执行计划
SET auto_explain.log_nested_statements = on;

-- 是否记录分析信息
SET auto_explain.log_analyze = on;

-- 是否记录 BUFFERS 信息
SET auto_explain.log_buffers = on;

-- 是否记录 timing 信息（需要开销）
SET auto_explain.log_timing = off;
```

### 1.4 永久配置

```bash
# kingbase.conf 添加
cat >> /data/kingbase/data/kingbase.conf << 'EOF'

# === AutoExplain 配置 ===
shared_preload_libraries = 'auto_explain'
auto_explain.enabled = on
auto_explain.log_min_duration = '1000'
auto_explain.log_analyze = on
auto_explain.log_buffers = on
auto_explain.log_nested_statements = on
EOF

# 需要重启数据库生效
./sys_monitor restart -D /data/kingbase/data
```

### 1.5 查看 AutoExplain 日志

```bash
# AutoExplain 的执行计划会写入数据库日志
tail -100 /data/kingbase/data/log/kingbase-$(date +%Y-%m-%d).log | grep "EXPLAIN"
```

## 二、PLSQL_Filter 存储过程过滤

### 2.1 简介

PLSQL_Filter 用于过滤和监控 PL/SQL 存储过程的执行，可以记录调用次数、执行时间等。

### 2.2 安装配置

```sql
-- 检查扩展
SELECT * FROM sys_available_extensions WHERE name = 'plsql_pldbgapi';

-- 安装（参考 plsql_pldbgapi 扩展）
CREATE EXTENSION IF NOT EXISTS plsql_pldbgapi;
```

### 2.3 使用 PL/SQL Debugger API

```sql
-- 启用 PL/SQL 调试
SET plsql.debug = on;

-- 使用 DBMS_PROFILER 分析 PL/SQL 性能
-- 先确保有相关扩展
CREATE EXTENSION IF NOT EXISTS plsql_plprofiler;
```

### 2.4 PLSQL_Filter 相关视图

```sql
-- 查看 PL/SQL 对象
SELECT
    routine_schema,
    routine_name,
    routine_type,
    data_type
FROM information_schema.routines
WHERE routine_schema NOT IN ('sys', 'information_schema')
ORDER BY routine_name;

-- 查看存储过程依赖
SELECT
    p.proname AS procedure_name,
    d.refobjid::regclass AS dependency
FROM sys_proc p
JOIN sys_depend d ON p.oid = d.objid
WHERE d.classid = 'sys_proc'::regclass;
```

## 三、NMON 系统监控

### 3.1 简介

NMON（Nigel's Monitor）是 IBM 开发的系统监控工具，可用于监控 Linux 服务器的 CPU、内存、磁盘、网络等资源。

### 3.2 安装 NMON

```bash
# CentOS/RHEL
yum install -y nmon

# 或下载安装
wget http://sourceforge.net/projects/nmon/files/nmon_x86_64_rhel6
chmod +x nmon_x86_64_rhel6
mv nmon_x86_64_rhel6 /usr/local/bin/nmon
```

### 3.3 交互式使用

```bash
# 启动 nmon（交互模式）
nmon

# 常用快捷键：
# c - CPU 监控
# m - 内存监控
# d - 磁盘监控
# n - 网络监控
# t - 进程监控
# q - 退出
```

### 3.4 数据采集模式（推荐用于巡检）

```bash
# 创建采集目录
mkdir -p /opt/nmon_data

# 每 60 秒采集一次，采集 1440 次（24 小时）
nmon -s 60 -c 1440 -f -m /opt/nmon_data -t

# 参数说明：
# -s 60    : 每 60 秒采集一次
# -c 1440  : 采集 1440 次
# -f       : 输出文件格式（.nmon）
# -m /path : 输出目录
# -t       : 包含 TOP 进程信息
```

### 3.5 Crontab 自动采集

```bash
# 每天早上 8 点开始采集
crontab -e

# 添加：
# 0 8 * * * /usr/local/bin/nmon -s 60 -c 1440 -f -m /opt/nmon_data -t
```

### 3.6 生成分析报告

```bash
# 下载 nmon 分析工具（nmonanalyser）
# 需要 Excel 或 LibreOffice

# 或使用命令生成摘要
# 查看采集的数据
cat /opt/nmon_data/*.nmon | grep "^CPU_ALL" | tail -1

# 查看内存使用
cat /opt/nmon_data/*.nmon | grep "^MEM" | tail -1
```

### 3.7 NMON 输出解读

```
CPU_ALL:
  User%   : 用户态 CPU 使用率
  Sys%    : 系统态 CPU 使用率
  Wait%   : I/O 等待 CPU 使用率
  Idle%   : 空闲 CPU 使用率
  CPU%    : 总 CPU 使用率

MEM:
  memtotal  : 总内存
  memfree   : 空闲内存
  buffers   : 缓冲区
  cached    : 缓存
  swapfree  : 交换区空闲
```

## 四、综合运维脚本

### 4.1 日常巡检脚本

```bash
#!/bin/bash
# KingbaseES 日常巡检脚本

KBS_HOME="/opt/Kingbase/ES/V9R2C14/Server"
LOG_DIR="/var/log/kingbase_inspection"
DATE=$(date +%Y%m%d_%H%M)

mkdir -p $LOG_DIR

echo "=== KingbaseES 日常巡检 $(date) ===" | tee -a $LOG_DIR/inspect_$DATE.log

# 1. 数据库状态
echo -e "\n--- 1. 数据库状态 ---" | tee -a $LOG_DIR/inspect_$DATE.log
$KBS_HOME/bin/sys_monitor status -D /data/kingbase/data 2>&1 | tee -a $LOG_DIR/inspect_$DATE.log

# 2. 连接数
echo -e "\n--- 2. 连接数统计 ---" | tee -a $LOG_DIR/inspect_$DATE.log
$KBS_HOME/bin/ksql -U system -d test -p 54321 -t -c "
SELECT '当前连接数: ' || count(*) || ' / ' || current_setting('max_connections')
FROM sys_stat_activity;
" 2>&1 | tee -a $LOG_DIR/inspect_$DATE.log

# 3. TOP SQL
echo -e "\n--- 3. TOP 5 慢 SQL ---" | tee -a $LOG_DIR/inspect_$DATE.log
$KBS_HOME/bin/ksql -U system -d test -p 54321 -t -c "
SELECT
    round(mean_exec_time::numeric, 2) || 'ms' AS avg_time,
    calls,
    round(total_exec_time::numeric, 2) || 's' AS total_time,
    substring(query, 1, 80)
FROM sys_stat_statements
ORDER BY mean_exec_time DESC
LIMIT 5;
" 2>&1 | tee -a $LOG_DIR/inspect_$DATE.log

# 4. 复制状态
echo -e "\n--- 4. 复制状态 ---" | tee -a $LOG_DIR/inspect_$DATE.log
$KBS_HOME/bin/ksql -U system -d test -p 54321 -t -c "
SELECT
    client_addr,
    state,
    pg_size_pretty(pg_wal_lsn_diff(sent_lsn, replay_lsn)) AS lag
FROM sys_stat_replication;
" 2>&1 | tee -a $LOG_DIR/inspect_$DATE.log

# 5. 表空间使用
echo -e "\n--- 5. 表空间使用 ---" | tee -a $LOG_DIR/inspect_$DATE.log
$KBS_HOME/bin/ksql -U system -d test -p 54321 -t -c "
SELECT
    spcname,
    pg_size_pretty(pg_tablespace_size(spcname)) AS size
FROM sys_tablespace;
" 2>&1 | tee -a $LOG_DIR/inspect_$DATE.log

# 6. 数据库大小
echo -e "\n--- 6. 数据库大小 ---" | tee -a $LOG_DIR/inspect_$DATE.log
$KBS_HOME/bin/ksql -U system -d test -p 54321 -t -c "
SELECT
    datname,
    pg_size_pretty(pg_database_size(datname)) AS size
FROM sys_database
ORDER BY pg_database_size(datname) DESC;
" 2>&1 | tee -a $LOG_DIR/inspect_$DATE.log

# 7. 锁等待
echo -e "\n--- 7. 锁等待情况 ---" | tee -a $LOG_DIR/inspect_$DATE.log
$KBS_HOME/bin/ksql -U system -d test -p 54321 -t -c "
SELECT
    l.locktype,
    l.relation::regclass,
    l.mode,
    a.usename,
    a.query
FROM pg_locks l
JOIN sys_stat_activity a ON l.pid = a.pid
WHERE NOT l.granted;
" 2>&1 | tee -a $LOG_DIR/inspect_$DATE.log

# 8. NMON 数据采集（30 秒采样）
echo -e "\n--- 8. 系统资源采集 ---" | tee -a $LOG_DIR/inspect_$DATE.log
nmon -s 30 -c 2 -f -m $LOG_DIR -t -N 2>&1 | tee -a $LOG_DIR/inspect_$DATE.log

echo -e "\n=== 巡检完成 ===" | tee -a $LOG_DIR/inspect_$DATE.log
echo "报告位置: $LOG_DIR/inspect_$DATE.log"
```

### 4.2 巡检 Crontab 配置

```bash
# 每天早上 8 点执行巡检
crontab -e

# 添加：
# 0 8 * * * /opt/scripts/kingbase_inspection.sh >> /var/log/inspection.log 2>&1
```

## 五、AutoExplain 日志分析

### 5.1 日志格式

```
2026-04-07 10:30:00.123 CST [12345]: [1-1] user=system,db=test LOG:  duration: 1234.567 ms  plan:
  Seq Scan on orders  (cost=0.00..12345.00 rows=1000 width=123)
    Output: id, customer_id, amount, create_time
    Filter: (create_time > '2026-01-01'::date)
```

### 5.2 识别问题 SQL

```bash
# 找出执行时间超过 5 秒的查询
grep "duration:" /data/kingbase/data/log/kingbase-*.log | \
  awk -F'duration: ' '{print $2}' | \
  awk '{if($1+0 > 5000) print}' | head -20
```

netstat -s      # 查看网络统计

# === KingbaseES ===
./sys_monitor status -D /data/kingbase/data

./ksql -U system -d test -p 54321 -c "\x"  # 切换 Expanded display
